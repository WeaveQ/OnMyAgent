# 开票字段与进度

## 开票批次卡

| 中文 | JSON | 说明 |
|------|------|------|
| 批次编号 | batchId | 可自拟 BILL-日期-序号 |
| 客户名称 | customerName | 开票抬头全称 |
| 客户编号 | customerId | 内部客户号（若有） |
| 票种 | invoiceType | vat_special（专票）/ general（普票）/ electronic（电子普票） |
| 运单数 | waybillCount | 本批含运单数 |
| 合计金额 | totalAmount | 含税/不含税需注明 |
| 税率 | taxRate | 13% / 9% / 6% / 3% / 1% 等 |
| 货物或服务名称 | itemName | 统一归类后的名称 |
| 账期 | accountPeriod | 月结 30/60/90 等 |
| 开票主体 | billingEntity | 客户当前有效开票主体 |
| 状态 | status | 见进度节点 |

## 进度节点（默认）

| 节点 | 说明 | 易拖原因 |
|------|------|----------|
| reconcile | 对账、运单确认 | 运单未回、金额异议 |
| material | 开票资料收集 | 业务未提供、抬头变更 |
| apply | 开票申请提交 | 资料不全反复退 |
| issuing | 开票中 | 财务排期、系统问题 |
| issued | 开票完成 | - |
| deliver | 寄送/交付 | 地址错误、签收未回 |
| received | 客户签收 | - |
| payment | 回款 | 账期内/逾期 |
| closed | 关闭 | - |
| disputed | 异议/核减 | 金额争议 |

## 进度表输出

| 节点 | 状态 | 计划/完成日 | 负责人 | 卡点 | 下次动作 |
|------|------|------------|--------|------|----------|

状态：`未开始` / `进行中` / `等待对方` / `完成` / `阻塞` / `逾期`

## 加速动作（示例）

- 对账阶段按客户分批，避免月末集中爆发
- 抬头信息建客户档案库，开票时直接调用，减少每次确认
- 开票申请一次备齐：客户信息 + 运单清单 + 金额，避免反复补充
- 寄送后跟踪签收，签收当日计入回款倒计时
- 账期过半主动软催一次，到期前 3 天硬催，逾期即升级
