# 定价焦虑方法论

> **何时加载**：用户问题包含 定价 / 几块钱 / 9.9 / 99 / 订阅 vs 一次性 / 免费 / 续费 / 转化率 / 客单价

**定价不是产品的事，是定位的事。** 改价格不能解决定位错的问题；定位对了，价格只是配套。

---

## 一次性 vs 订阅决策树

绝大多数一人公司应该**先做一次性付费**，验证后再考虑订阅。

```
用户使用频率高（每周 1+ 次）？
├── 是 → 用户用完会有持续数据 / 信任沉淀？
│   ├── 是 → 订阅可行（但仍建议先一次性验证 6 个月）
│   └── 否 → 一次性
└── 否 → 一次性（订阅会被高流失率拖死）
```

**陷阱**：用户**说**会持续用，但行为数据告诉你不会。**先按行为数据判断，不按用户嘴**。

## 价格阶梯 + 购买理由强度

| 价格区间 | 适合什么 | 用户预期 | 一人公司可行度 |
|---|---|---|---|
| **9.9 元** | 试试看 / 单次诊断 / 一份模板 | 即时小价值 | ✅ 容易跑动，但单笔利润低 |
| **19.9 - 29.9** | 完整报告 / 单次手工服务 | 一份具体交付 | ✅ 适合 MVP 验证 |
| **49 - 99** | 解决一个明确问题 / 1 小时咨询 | 看得见的结果 | ✅ 高 ROI 区间 |
| **99 - 199** | 1 对 1 深度诊断 / 完整 setup | 专业服务 | ✅ 适合服务型 |
| **299 - 499** | 陪跑一周 / 完整咨询 | 持续支持 | ⚠️ 需要时间投入 |
| **999 - 1999** | 月度服务 / 高端订阅 | 强 ROI 证据 | ⚠️ 客户少，要细心维护 |
| **5000+** | 项目制 / 企业服务 | 复杂交付 | ❌ 一人难以规模化 |

**普通用户对价格的真实容忍度**：

- 9.9：试错成本（无脑买）
- 49：需要一句话理由
- 99：需要看到结果案例
- 299+：需要 1 对 1 沟通

## 中文区低价订阅陷阱

**9.9 元/月不是护城河，是陷阱。**

中文区低价订阅最容易吸进来的是：
- 随手试试的用户（流失率高）
- 比价用户（一有更便宜的就走）
- 薅羊毛用户（投诉率高）

成本上：
- 客服 / 投诉成本 / 退款率高
- API 成本 + 流量成本叠加 → 净利润可能为负
- 续费率低（中文区订阅文化不成熟）

**中文区的合理定价节奏**：

1. 先用一次性 9.9 / 19.9 / 49 验证有人愿意付钱
2. 再用 99 / 199 验证愿意付高
3. 最后才考虑月度订阅（且通常 49 / 99 月起）

## 购买理由强度评分（D7）

定价之前先评估购买理由：

| 维度 | 强信号 | 弱信号 |
|---|---|---|
| **风险** | 用错有具体后果（被扣钱、被骂、错过 deadline） | 没用也行 |
| **焦虑** | 这件事一直让用户睡不着 | 偶尔想起一下 |
| **重复性** | 每周 / 每月反复发生 | 一年一两次 |
| **意愿** | 用户主动说"我现在就要" | "有空再看" |

四项全拉满 → 高价位也可行（参考 Citely）
四项中只有 1-2 项 → 只能低价位
四项全无 → 不要做这个产品

## 涨价不是产品的事，是定位的事

错误做法：
- 觉得 9.9 不赚钱 → 涨到 99
- 用户不买就降回 9.9

正确做法：
- 9.9 不赚钱 = 你的产品定位是"工具"（多个低价用户）
- 想做 99 → 重新定位为"咨询 / 服务"（少量高价用户）
- **用户不一样，产品形态不一样**——不是同一个产品涨价

---

## 怎么从单笔 100 试到订阅

不是直接上 SaaS。**阶梯**：

1. **手工服务 99 元/次**（验证有人愿意为这事付钱）
2. **服务化模板 49 元/次**（沉淀流程，一个人能做更多单）
3. **半自动化工具 29.9 元/次**（核心环节工具化，仍然按次付）
4. **订阅 99 元/月**（用户每月用 3+ 次时再做订阅）
5. **完整 SaaS 199 元/月**（真有规模化交付能力时）

> **不要跳级。跳了的代价是用户基础不稳，定价失去说服力。**

---

## 危险信号

- "我先免费做大用户量再变现" → 一人公司最大幻觉
- "9.9 月费薄利多销" → 中文区不成立
- "等用户说'你应该收费'再收费" → 用户永远说免费好
- "竞品收 99 我收 49" → 价格战是死路
- "我加更多功能就能涨价" → 加错功能反而降价

---

## 收尾必做

让用户**今晚**：

1. 选一个具体的人（具体人卡片里那个人）
2. 把价格按上面阶梯定一个起点（多数情况选 49 / 99）
3. 直接发消息："这个东西，[价格] 你下单不？"

> **不要做完产品才问价格。先问价格，再做产品。**
