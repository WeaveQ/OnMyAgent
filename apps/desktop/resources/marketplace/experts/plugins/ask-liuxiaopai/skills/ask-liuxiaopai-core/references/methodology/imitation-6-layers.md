# 像素级模仿 6 层方法论

> **何时加载**：用户问题包含 抄 / 模仿 / 借鉴 / 像 X 一样做 / 复刻 / 做个差不多的

> **像素级模仿的真正目的，不是让你学会抄，而是理解一整条"问题→解法→结果→付费"的闭环链路。**

绝大多数"抄"失败的根源：抄了 UI 和功能，没抄"为什么有人付钱"。差距不在代码，**差距在购买理由**。

---

## 必要条件 vs 充分条件

对方赚钱 = 必要条件 + 充分条件

- **充分条件**：UI 漂亮、功能多、技术先进、设计精良——可见的、可抄的
- **必要条件**：分发、品牌、timing、信任、社群、个人 IP——不可见的、抄不走的

**只抄充分条件，结果是做出一个"看起来一样但没人用"的产品**。

---

## 6 层拆解（按重要性，从底到顶）

要"像素级"理解一个赚钱产品，按这 6 层拆，**每层都要写清楚**：

### 第 1 层：解决的问题（最关键）

不是"它做了什么功能"，是"用户用之前在烦什么"。

要写出：
- 用户在用这个产品**之前 5 分钟**，遇到的具体场景
- 用户**为什么会去搜**（or 谁推荐的）
- **焦虑**是什么（不是功能名，是情绪）

举例：Citely（学术引文工具）解决的不是"格式化引文"——是"博士生交论文前一天发现引文格式不对的崩溃"。

### 第 2 层：用户路径（onboarding）

从用户**首次进入** → **明白了对我有用** 的最短时间。

要写出：
- 第一屏看到什么
- 多久能完成一次有意义的操作
- 哪一步用户最容易流失

> 大多数人在 90 秒内决定要不要继续。**这 90 秒怎么设计的？**

### 第 3 层：结果交付（aha moment）

用户付钱前必须看到的"具体结果"。

不是抽象承诺，是：
- 一份可下载的 PDF 报告
- 一个具体的分数
- 一张可分享的图
- 一段可复制的文本

> **没有 aha moment 的产品不会被付费。**

### 第 4 层：商业化设计

什么免费、什么收费、为什么这么卡。

要写出：
- 免费用到哪一步会卡住
- 卡的位置是不是用户**最想继续** 的位置
- 第一次付费多少钱（一次性 / 订阅 / credits）
- 续费的钩子是什么（社交锁定 / 数据沉淀 / 习惯）

### 第 5 层：流量入口

用户怎么发现这个产品。

要写出：
- 主流量来源（SEO / 内容 / 榜单 / 付费 / 社群 / 推荐）
- 每个来源的占比（看 SimilarWeb / 评论区 / 推荐链接）
- **谁在替它免费推荐**（KOL / 媒体 / 满意用户）

### 第 6 层：技术闭环（最不重要）

最后才看代码。

要写出：
- 大致用什么栈
- 哪部分用了 AI / 哪部分是手工
- 成本结构（API / 服务器 / 人力）

> **代码是最容易抄的，所以也是最不值钱的。**

## 怎么拆一个赚钱产品（实操）

不要看官网，看这 5 个地方：

1. **Twitter / X 上对方账号的回复区**——用户原话
2. **Reddit / Hacker News 的讨论**——挑刺也是数据
3. **App Store / Product Hunt 评论区**——付费用户最真实
4. **官网 testimonial**——cherry pick，但能看出他们认为哪些角度最值钱
5. **取消订阅页 / 退费政策**——他们最怕用户因为什么走

把上面 5 个地方的原话整理成一份"真实使用证据"，然后**反推 6 层**。这才是模仿。

---

## 模仿的 3 个产物

模仿之后必须输出：

1. **一份 6 层拆解笔记**（每层至少 5 条具体观察）
2. **3 个差异点**（你打算做的版本和原版的 3 个区别——必须比原版强，不只是"差不多"）
3. **一句话主张**（用模仿对象学到的"购买理由"，写出**你的产品**的购买理由）

只复制 UI 和功能 = 失败的模仿。复制"购买理由的结构" + 注入你的差异 = 真正的像素级模仿。

---

## 危险信号

- "我看了它的代码，技术上不难" → 你抄错了层
- "我做个一样的功能" → 没拆其余 5 层
- "他们做得不好的地方我能改进" → 改进的是"看得见的部分"，不是必要条件
- "等我做出来再考虑流量" → 假设产品自动有流量，第 5 层完全没设计

---

## 收尾必做

让用户回去做一件事：

> **今晚选一个赚钱产品，按 6 层拆一份笔记。下次我们一起看你的笔记，找出你能复用的"必要条件"。**

不让用户停在"我要不要抄"——让他做"6 层拆解"这个具体动作。
