# 案件字段与进度

## 案件卡

| 中文 | JSON | 说明 |
|------|------|------|
| 案件编号 | caseId | 可自拟 CLAIM-日期-序号 |
| 运单号 | waybillNo | 必填优先 |
| 线路 | lane | 装→卸 |
| 货品 | cargo | 品名、件数、声明价值 |
| 异常类型 | type | damage_outer / damage_inner / wet / delay / loss / other |
| 发现时间 | discoveredAt | |
| 发现地点/环节 | discoveredAtNode | 装/卸/在途/签收后 |
| 初步原因（主张） | allegedCause | 各方说法分开记 |
| 客户要求 | customerDemand | 金额/重发/道歉等 |
| 保价/保险 | cover | 有无、保司名、保单号（知多少写多少） |
| 现状 | status | 见进度节点 |

## 时间线（按序）

提货/装车 → 发车 → 中转 → 到站 → 卸货/签收 → 客诉 → 报案 → …

知多少写多少；未知标 `未知`。

## 进度节点（默认）

| 节点 | 说明 | 易拖原因 |
|------|------|----------|
| intake | 收案、材料初筛 | 信息不全 |
| evidence | 固证、补证 | 现场已清、监控过期 |
| report_insurer | 向保司/平台报案 | 材料不合格反复退 |
| assessment | 定损 | 等公估/等报价 |
| supplement | 补充材料 | 清单不清 |
| negotiate | 谈判/方案 | 责任扯皮 |
| settle | 赔付/结案 | 付款流程 |
| closed | 关闭 | — |
| withdrawn | 撤销 | — |

## 进度表输出

| 节点 | 状态 | 计划/完成日 | 负责人 | 卡点 | 下次动作 |
|------|------|--------------|--------|------|----------|

状态：`未开始` / `进行中` / `等待对方` / `完成` / `阻塞`

## 加速动作（示例）

- 监控申请写清时间窗与摄像头位置，赶在覆盖周期前  
- 定损前备好购置发票/维修报价/同类市价截图  
- 每周固定催办日 + 书面（微信）留痕  
- 客户侧先给「处理节奏」降低升级频率，同时内部不停补证
