# 库存台账字段与模板

## 变动类型 movement.type

| 值 | 中文 | 账面影响 |
|----|------|----------|
| inbound | 入库 | +数量 |
| outbound | 出库 | −数量 |
| transfer | 移库 | 源货位− / 目标货位+ |
| stocktake_gain | 盘盈 | +数量（须备注原因） |
| stocktake_loss | 盘亏 | −数量（须备注原因） |
| adjust | 调整 | ±−（仅纠错，写清依据） |

## 变动行字段

| 中文 | JSON | 说明 |
|------|------|------|
| 时间 | at | 操作或登记时间 |
| 类型 | type | 见上表 |
| 运单号 | waybillNo | 主键优先 |
| 货主/客户 | shipper | 可选 |
| 货品 | sku / cargoName | 品名简述 |
| 数量 | qty | 数字 |
| 单位 | uom | 件/箱/托/吨/方 |
| 货位 | bin | 如 A-03 |
| 移库目标货位 | binTo | 仅 transfer |
| 操作人 | operator | 仓管/司机 |
| 关联单据 | docRef | 交接单号等 |
| 来源 | source | 微信/表/口述 |
| 备注 | remarks | 破损、待确认等 |

| 单位 | uom | |

| 滞留天 | dwellDays | 相对今天 |
| 状态 | status | in_stock / partial_out / pending_confirm / anomaly |

## 主键建议

1. `waybillNo + bin + uom`  
2. 无运单时 `cargoName + bin + 批次备注`（弱键，易冲突须提示）  
3. 对不上 → **待确认**，禁止硬凑

## 当日简报模板

```markdown
【仓储简报】{日期} {网点}

### 进销存
入 {n} 笔 / 出 {n} 笔 / 移 {n} 笔 / 盘点差异 {n}

### 异常
{列表或「无」}

### 滞留 Top
| 运单 | 货位 | 滞留天 | 备注 |

### 待确认
{缺字段条目}

### 关键节点提醒
- 卸货是否当面确认件数
- 出库是否复核
- 是否存在回单/出库未闭环导致账面虚高
```

## 默认滞留阈值

- 预警：在库 **> 7 天**（用户可改）  
- 紧急：在库 **> 15 天** 或用户指定
