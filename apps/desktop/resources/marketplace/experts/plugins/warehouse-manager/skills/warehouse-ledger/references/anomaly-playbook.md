# 库存异常与倒查

## 异常类型

| 类型 | 信号 | 优先倒查 |
|------|------|----------|
| QTY_MISMATCH | 账面 ≠ 实物/单据件数 | ① 入库清点 ② 出库复核 ③ 是否重复登记 |
| NEGATIVE_STOCK | 账面 < 0 | 漏入库、重复出库、运单串票 |
| WRONG_BIN | 货在 A 账在 B | 移库未登记、贴错标签 |
| DOC_VS_CARGO | 运单件数 ≠ 交接/现场 | 卸货未当面确认、中途甩货 |
| OVERSTAY | 超滞留阈值仍在库 | 调度未派、客户未提、回单/放货未闭环 |
| GHOST_STOCK | 账有货现场无 / 虚高 | 已出库未销账、回单未回收当仍在库 |
| SUSPECT_MISSHIP | 错发/漏发投诉或装车疑点 | 出库双人复核、装车清单、门岗放行 |
| DAMAGE | 破损记录 | 入库是否已注残、责任未定是否冻结出库 |

## 倒查顺序（通用）

1. **单据链**：运单 → 入库交接 → 出库装车清单 → 司机签收  
2. **时间线**：最后一次货动是谁、何时、哪一货位  
3. **人**：当班仓管、装卸工、提货司机  
4. **现场**：拍照、货位标签、残留包装  

## 盘点差异处理

| 差异 | 动作 |
|------|------|
| 盘盈 | 记 stocktake_gain，备注发现人与可能原因；查是否漏入库 |
| 盘亏 | 记 stocktake_loss，标红；查是否漏出库/盗损/错发 |
| 暂不能定 | 冻结该运单出库建议，进「待确认」 |

## 错发漏发检查项（出库）

- [ ] 运单号与货签一致  
- [ ] 件数/托数与账面一致  
- [ ] 货位与拣货一致  
- [ ] 第二人复核或扫码/勾单  
- [ ] 司机交接签字/照片  

## 入库清点检查项

- [ ] 当面清点件数（卸货未当面确认 → 高风险）  
- [ ] 破损/潮湿当场记录  
- [ ] 货位分配并登记  
- [ ] 与运单吨方/件数比对  

## 虚高库存提醒话术（给调度）

```
调度旁，仓里这些票账面还在，但可能已提走未销账或回单未闭环，请协助确认是否已出库：
1）{运单} {货位} 滞留{天}
…
确认后我这边改出库流水，避免库存虚高。
```
