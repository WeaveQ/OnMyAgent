# 三档方案：最快 / 最便宜 / 平衡

默认在同一询价下给三档；用户只要一档时仍简要说明其它档取舍，便于砍价时切换。

## 档位定义

| 档位 | 目标 | 常见手段 | 风险 |
|------|------|----------|------|
| **最快** | 时效与确定性优先 | 整车/优质专线、优先提送、少中转 | 价格高、空驶与放空风险 |
| **最便宜** | 总成本优先 | 拼载/普通专线、自送自提、放宽时效、减附加服务 | 时效波动、服务投诉 |
| **平衡**（默认推荐） | 时效可接受 + 利润健康 | 主流专线+必要提送、附加费单列清楚 | 需明确不含项 |

## 对比表模板

| 档位 | 方式与时效 | 参考合计 | 已含 | 不含/降级 | 适合 |
|------|------------|----------|------|-----------|------|
| 最快 | | ¥…（示意/实算） | | | 急单、高货值 |
| 平衡 | | | | | 多数客户 |
| 最便宜 | | | | | 预算紧、可等 |

## 生成规则

1. 三档 **费用项列名一致**，用金额或「—」区分，避免三套完全不同的表头。  
2. 从最快 → 最便宜，优先减 **服务与时效**，不是无说明狂砍干线。  
3. 每档 2–4 条「为什么这样报」；平衡档写清为何默认推荐。  
4. 若硬件约束（必须冷藏/尾板/危化）导致无法做「最便宜」减配，说明该档不可用或仅有限度降价。  
5. 无成本数据时：三档都可给 **结构 + 相对关系**（如最快约比平衡高 15–30%·示意），禁止假装精确市价。

## 与砍价的衔接

客户嫌贵 → 引导改选 **最便宜** 或 **平衡减项**，并输出对应话术；对内提示是否触碰底价。
