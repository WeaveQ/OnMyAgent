# 询价字段

## 必填（缺则难出可靠价）

| 中文 | JSON | 说明 |
|------|------|------|
| 提货地 | origin | 尽量到区/镇 |
| 卸货地 | destination | 尽量到区/镇 |
| 重量或体积 | cargo.weightT / cargo.volumeCbm | 至少一项；泡重都有更好 |
| 时效要求 | sla | 如 48h、T+1、普通 |
| 运输方式/车型意向 | mode | 专线/整车/零担；4.2/6.8/9.6 等 |

## 强烈建议

| 中文 | JSON | 说明 |
|------|------|------|
| 货品与包装 | cargo.name / packaging | 普货/泡货/重货、可否叠压 |
| 提送要求 | service.pickup / delivery | 门到门、自送网点等 |
| 装卸 | service.loading | 含装/含卸/仅运输/需叉车 |
| 特殊要求 | special | 尾板、进仓、预约、冷链、危化 |
| 装货时间窗 | readyAt | 今天/明天/等通知 |
| 回单 | pod | 电子/原件/加急 |
| 是否含税 | taxIncluded | 含税价或未税+专票 |
| 保价 | insurance | 货值或是否保 |
| 历史成交参考 | historyRate | 你们指导价或上次价 |
| 客户合作备注 | accountNotes | 老客户/回款/走量（有才写） |

## 信息过少时的处理

1. 列出缺失必填项（编号，一次问清）。  
2. 若用户要求「先给个大概」：输出 **结构空表 + 假设**（例如假设可叠、按方计、不含提送），所有数字标 `示意`。  
3. 禁止在关键字段全空时给出「精确到个位的一口价保证」。

## 追问模板

```
收到询价，先确认这几项好报准：
1）提货/卸货详细到区或地址类型（工厂/园区/住宅）
2）吨和方分别多少？可否叠压？
3）要专线还是整车？车型有要求吗？
4）时效：当天/次日/48h/普通？
5）是否含提货、送货、装卸、尾板、进仓、回单（电子/原件）、保价？是否含税？
6）有无你们历史指导价或上次成交价？（便于对齐、沉淀）
```
