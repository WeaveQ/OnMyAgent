# 对账主表、差异与原因码

## 默认规则（输出时写明，用户可改）

| 规则 | 默认 |
|------|------|
| 差异定义 | 对方金额 − 我方应付 |
| 标红阈值 | \|差异\| > ¥50 |
| 大额待拍板 | 单票 \|差异\| > ¥2000 或用户指定 |
| 回单卡付 | 回单未齐（pending/partial/disputed）建议「暂缓」或按约定比例 |
| 币种 | CNY，两位小数 |

## 对账明细主表

| # | 运单号 | 线路 | 费用分项摘要 | 我方应付 | 对方金额 | 差异 | 回单状态 | 原因初判 | 建议 |
|---|--------|------|--------------|----------|----------|------|----------|----------|------|

### 建议枚举

- `建议付`：匹配、差异≤阈值、回单满足结账条件  
- `建议暂缓`：回单未齐 / 争议 / 待查证  
- `建议追差`：对方多计或我方漏记需向对方确认  
- `建议补录`：我方漏记附加费等  
- `无法匹配`：键对不上  

## 汇总块

- 票数 / 匹配数 / 差异票数 / 标红数 / 回单未齐数  
- 建议付合计 / 建议暂缓合计 / 建议追差合计  
- 回单 P0/P1/P2 数量  

## 无法匹配区

| 来源 | 关键信息 | 说明 |
|------|----------|------|
| 对方账单 / 我方表 | 单号或摘要 | 缺键或冲突 |

## 原因码（受控，仅用下列；不确定用 `待查证`）

| 码 | 中文 | 典型场景 |
|----|------|----------|
| RATE_DIFF | 单价/议价不一致 | 双方费率表不同 |
| EXTRA_FEE | 附加费争议 | 放空、等候、进场、多点 |
| MISSING_FEE | 我方漏记 | 有约定未进表 |
| DUP_BILL | 重复计费 | 同票两次出现 |
| WRONG_WAYBILL | 串票/错票 | 单号相近或线路张冠李戴 |
| POD_HOLD | 回单未齐卡结 | 对方已请款但我方未收回单 |
| POD_FORM | 回单形态不符 | 只要原件却只有照片 |
| QTY_WEIGHT | 吨方/件数不符 | 过磅或点件差 |
| REBATE | 回扣/信息费口径 | 含或不含争议 |
| ROUNDING | 四舍五入/币种 | 小额差 |
| PERIOD_CUT | 账期切割 | 跨月归属 |
| PENDING_PROOF | 待查证 | 信息不足，禁止猜金额 |
