# 运力池标准字段

## 主键规则

| 优先级 | 键 | 说明 |
|--------|----|------|
| 1 | plateNo | 车牌号，最稳定 |
| 2 | driver.phone | 无车牌时用司机手机 |
| 3 | driver.name + vehicle.type | 仅姓名时需结合车型，冲突则拆条或追问 |

## 核心字段（维护清单建议齐全）

| 中文字段 | JSON 字段名 | 说明 | 格式 / 校验 |
|---------|------------|------|-------------|
| 车牌号 | plateNo | 车辆唯一标识 | 新能源/普通车牌格式尽量规范 |
| 司机姓名 | driver.name | 联系人 | 非空更佳 |
| 司机电话 | driver.phone | 联系电话 | 11 位手机或带区号座机 |
| 当前位置 | location | 城市+区/镇/地标 | 至少到区/镇；仅“在路上”需追问 |
| 车型 | vehicle.type | 如 4.2 厢 / 6.8 高栏 / 9.6 平板 | 规范车长+厢型 |
| 额定载重 | vehicle.ratedLoadT | 吨 | 数字 |
| 额定方数 | vehicle.ratedVolumeCbm | 立方米 | 数字 |
| 剩余载重 | residual.loadT | 还能拉多少吨 | 数字；“还能拉一点”必须追问 |
| 剩余方数 | residual.volumeCbm | 还能拉多少方 | 数字 |
| 司机/车辆状态 | status | 见状态字典 | 枚举 |
| 预计空闲时间 | freeAt | 何时可接下一票 | 明确时间点或“已空闲” |
| 可接方向 | preferredLanes | 愿意去的方向/城市 | 列表；“随便”记 any |
| 特殊能力 | capabilities | 尾板、冷藏、危化资质、装卸工等 | 列表 |
| 更新时间 | updatedAt | 本条信息时间 | ISO 或本地时间字符串 |
| 信息来源 | source | 微信群/私聊/电话/表格 | 备注 |
| 新鲜度 | freshness | fresh / aging / stale | 由更新时间计算 |
| 备注 | remarks | 习惯、表现提示等 | 可选 |

## 选填 / 辅助字段

| 中文字段 | JSON 字段名 | 说明 |
|---------|------------|------|
| 所属车队/外请 | fleet | owned / hired / partner |
| 当前任务摘要 | currentJob | 在途票简述（到哪卸、大约几点） |
| 历史习惯摘要 | habits | 常跑线、响应习惯（仅有依据时填写） |
| 表现备注 | performanceNotes | 准点/拒单等，禁止无依据打分 |

## 新鲜度阈值（默认可由调度改）

| 状态 | 默认条件 | 展示 |
|------|----------|------|
| fresh | 更新时间 ≤ 2 小时 | 正常可用 |
| aging | 2 小时 < 更新 ≤ 6 小时 | 黄色提醒，推荐时注明“信息偏旧” |
| stale | 更新时间 > 6 小时，或明确“已下班/已接单”后未再确认 | 标红，不作为当前可用，进入催更名单 |

> 夜班/长途场景可将阈值放宽，但必须在输出中注明所用阈值。

## 字段置信度

- **高**：原文明确，格式合规
- **中**：有表述但模糊（“大概还能拉半车”“晚点卸完”）
- **低**：识别存疑或仅靠历史习惯推断，必须请调度/司机确认

## 运力池条目 JSON 示例

```json
{
  "plateNo": "粤B·D12345",
  "driver": { "name": "张师傅", "phone": "13800000000" },
  "location": "东莞茶山",
  "vehicle": { "type": "6.8米高栏", "ratedLoadT": 10, "ratedVolumeCbm": 45 },
  "residual": { "loadT": 8, "volumeCbm": 30 },
  "status": "empty_idle",
  "freeAt": "已空闲",
  "preferredLanes": ["东莞本地", "深圳"],
  "capabilities": ["尾板"],
  "updatedAt": "2026-07-20T14:30:00+08:00",
  "source": "微信群",
  "freshness": "fresh",
  "remarks": ""
}
```
