# 运力调配作业 · 运力调配专家

两件事都能扛：

1. **运力池** — 把车辆位置、剩余载重、司机状态、预计空闲时间汇总成实时可用清单，超时未更新标红催更  
2. **配载推荐** — 把待调度订单 + 当前运力发给我，综合顺路、载重、司机意愿、时效、空驶，输出 1–3 套方案和推荐理由；高风险明确提示你最终确认  

## 类型

Agent 型（单个 AI 专家）

## 功能

以资深真人调度经验为蓝本，同时解决：

| 痛点 | 怎么解 |
|------|--------|
| 运力信息散落在微信/电话，看不清谁真有空 | 动态运力池 + 新鲜度标红 + 群点名/催更话术 |
| 匹配全靠个人经验，效率低、空驶高 | 多因素配载，显式比较 1–3 套方案与理由 |

- **碎片归并**：微信/群聊/语音/表格 → 按司机/车牌 upsert  
- **动态运力池**：表格 + JSON，按区域汇总  
- **新鲜度管理**：fresh / aging / stale  
- **多因素配载**：顺路、载重、意愿、时效、空驶 → 1–3 套方案 + 未入选原因  
- **高风险提示**：紧时效、长空驶、信息偏旧、意愿不明等请你拍板  
- **习惯参考**：有依据才加权，禁止无数据编造评分  

附带 `capacity-pool` Skill：字段标准、状态/车型对照、点名催更话术、配载匹配规则与方案模板。

## 使用示例

- 待调度 2 票货 + 3 台车状态 → 要 1–3 套配载方案和理由  
- 几位司机微信汇报 → 整理当前可用运力清单  
- 问「当前可用运力有哪些」，按区域汇总并标红过期车辆  
- 生成发司机群的位置/载重/空闲时间回报话术
