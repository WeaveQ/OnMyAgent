# 账龄与催收节点

## 账龄（agingDays）

`agingDays = 今天 − dueDate`（未到期为负或「距到期 X 天」）。

| 分段（示例） | 含义 |
|--------------|------|
| 未到期 | open / due_soon |
| 1–7 天 | 逾期初期 |
| 8–30 天 | 逾期加深 |
| 31–60 天 | 高风险 |
| 60+ 天 | 坏账关注（对内） |

用户可改分段。

## 默认催收节点（可配置）

| 节点 | 触发 | 默认话术档 |
|------|------|------------|
| D-7 | 到期前 7 天 | 礼貌提醒 |
| D-3 | 到期前 3 天（可选） | 礼貌提醒加强 |
| due | 到期当天 | 正式提醒付款 |
| +3 | 逾期 3 天 | 正式催告 |
| +7 | 逾期 7 天 | 催告 + 对内升级关注 |
| +15 | 逾期 15 天 | 升级施压骨架 |
| +30 | 逾期 30 天 | 对内停运/法务建议清单 |

## 长账期与承兑

- net 60/90：台账标 `long_terms`；D-7 仍要做，避免「太远就忘」。  
- 承兑：到期日用 **票面到期** 另列；现金账龄与票期分开看。  
- 多次「下周一定」「票在路上」：记入 lastFollowUp，连续 N 次无实质进展 → 提高力度档。

## 优先级排序（今日清单）

1. 大额逾期 + 劣迹回款  
2. 逾期久  
3. 到期当天 / D-7 大额  
4. 小额临近
