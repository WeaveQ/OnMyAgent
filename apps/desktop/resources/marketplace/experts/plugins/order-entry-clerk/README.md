# 物流单录入作业 · 物流单专家

把客户发来的微信文字、语音、图片或单据模板发给我，我会先确认模板，再用 HTML 生成一张真实的物流单效果图。你补充一次信息，单据就原位完善一次，直到定稿交付。

## 类型

Agent 型（单个 AI 专家）

## 功能

以资深真人调度员经验为蓝本，解决"订单信息碎片化、录单慢、易错漏，信息不全导致反复确认"的痛点：

- **多模态识别**：客户常发语音或拍照，支持文字、语音转写、手写单/便签照片识别
- **模板优先**：先问是否有指定模板，有就按模板生成，无就按通用格式生成
- **真实单据效果**：对话中持续更新同一份 HTML 单据，已知值和待补项都在实际格子中展示
- **通用单据支持**：主要生成物流单，也可按需生成发货单、发车单/派车单和运单
- **核心字段快速提取**：托运方、收货方、起讫地、货物、数量/包装/重量/体积、提货/到达时间、车辆与特殊要求
- **缺失检查与智能追问**：对照必填字段逐项检查，生成礼貌、一次问清、可直接转发的追问话术
- **追问话术中转**：缺失信息合并成一条礼貌话术，你确认后转发客户，客户回复转回我补全
- **客户历史习惯参考**：老客户自动带出常用地址、联系人、结算方式、车型偏好，标注来源请确认
- **完整单据交付**：默认交付可打印 HTML，环境支持时同源导出 PDF；Excel 或系统导入格式仅按需附加

附带 `order-entry` 知识 Skill，内含订单字段标准、常见货物描述与特殊要求表达对照、分场景追问话术模板。

## 使用示例

- 客户微信发来："明天上午从深圳发两台注塑机到东莞，收货人刘经理，要尾板车，越快越好"，帮我生成物流单并逐步补全效果图
- 这张手写发货单照片有些潦草，帮我识别内容，并列出还需要向客户确认哪些信息
- 帮我做一张派车单，先问我有没有公司模板，没有就按通用格式
- 帮我汇总今天收到的所有订单，检查有没有漏单或信息不全的，生成待追问清单
