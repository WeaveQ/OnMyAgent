# 通用物流单字段

本字段集用于无指定模板的普通货运场景。它是业务通用格式，不是法定统一样式；危险货物、国际运输、多式联运等使用对应专用模板。

## 固定版式契约

无指定模板时，字段必须按 `assets/logistics-waybill-template.html` 的顺序和表格结构呈现：

1. 标题、副标题、单号、起讫地、发货日期
2. 托运单位/姓名/电话与收货单位/姓名/电话双栏
3. 货物名称、件数、包装、重量/体积、保险金额、保险费、代收货款、备注
4. 车牌号、家庭住址、驾驶证号、司机姓名与电话
5. 到付/已付/回付/欠付、提货自提/送货
6. 金额大写及数字金额
7. 发货单位、承运司机、本部经手人、收货单位四方签字日期
8. 一联存根、二联收货单位、三联发货单位说明及页脚地址电话

模板外业务信息统一放入“备注”，不得新增版块或改变字段顺序。

## 必填字段（缺失必须追问）

| 中文字段 | JSON 字段名 | 说明 | 格式校验 |
|---------|------------|------|---------|
| 托运方名称 | shipper.name | 发货公司或个人 | 非空 |
| 托运方联系人与电话 | shipper.contact / shipper.phone | 装货现场可联系人 | 姓名非空；电话格式与用户所在地区相符 |
| 装货地址 | shipper.address | 起运地及详细提货点 | 至少细化到区/县，现场交接时应有门牌或地标 |
| 收货方名称 | consignee.name | 收货公司或个人 | 非空 |
| 收货方联系人与电话 | consignee.contact / consignee.phone | 卸货现场可联系人 | 姓名非空；电话格式与用户所在地区相符 |
| 卸货地址 | consignee.address | 目的地及详细交付点 | 至少细化到区/县，现场交接时应有门牌或地标 |
| 货物名称 | cargo.name | 品名，需规范表述 | 非空，禁用"东西""货"等模糊词 |
| 数量/件数 | cargo.quantity | 件数或台数 | 数字 + 单位 |
| 重量或体积 | cargo.weight / cargo.volume | 计价与配载依据，至少一项 | 重量 kg/吨；体积 立方米/方 |
| 计划提货时间 | timeline.pickup | 期望装货/发运时间 | 明确日期与大致时段 |
| 要求到达时间 | timeline.delivery | 到达时限 | 明确日期、时间点或可验证时效 |

## 选填字段（缺失可后补，但建议提示）

| 中文字段 | JSON 字段名 | 说明 |
|---------|------------|------|
| 车型要求 | vehicleRequirement | 如 4.2 米厢车、7.6 米、9.6 米、尾板车、高栏、平板、冷藏车 |
| 特殊要求 | specialRequirements | 易碎、防潮、冷链温控、回单、送货上门、卸货协助、等通知放货等 |
| 货物价值 | cargo.declaredValue | 涉及保价，高价值货物建议提示保价 |
| 包装方式 | cargo.packaging | 木箱、托盘、纸箱、裸装等 |
| 货物规格/尺寸 | cargo.dimensions | 单件或总体长宽高，大件货建议必填 |
| 结算方式 | paymentMethod | 现付、到付、月结、回单付 |
| 运费与附加费 | charges | 运费、装卸费、送货费等；未确认时留空 |
| 是否保价 | insurance | 保价金额 |
| 提货/交付方式 | handover | 上门提货、送货上门、站点自提等 |
| 承运方 | carrier.name | 承运企业或个体承运人 |
| 车辆与司机 | carrier.vehicle / carrier.driver | 车牌、车型、司机姓名和电话；派车前可留空 |
| 回单要求 | receiptRequirement | 电子回单、纸质回单、返回时限 |
| 备注 | remarks | 其他补充信息（见下方「备注精简规则」） |

## 字段抽取铁律（精准映射，禁止一锅端）

从用户/客户日常对话抽取字段时，**先拆到专用字段，再考虑备注**。禁止把整段说明、复述摘要或已映射字段原样塞进 `remarks`。

| 对话内容类型 | 正确落点 | 错误做法 |
| --- | --- | --- |
| 提货方式（上门提货/自提/送货） | `handover` | 写入备注 |
| 结算（到付/现付/欠付/月结…） | `payment.method`；月结细节可一句备注 | 整段结算说明堆备注 |
| 保价/货值 | `cargo.declaredValue` / 保险相关字段 | 写成长备注 |
| 回单要求 | 优先 `remarks` 中**一句**回单要点，或专用回单字段若协议有 | 复述整段回单故事 |
| 计划提货 / 要求到达 | `timeline.pickup` / `timeline.delivery` | 再抄进备注 |
| 车型要求 | `vehicleRequirement` | 再抄进备注 |
| 货物件数/包装/重量/尺寸 | `cargo.*` | 用备注代替货物行 |
| 装卸现场联系人电话 | 装货侧 → `shipper.contact/phone`；卸货侧 → `consignee.*` | 仅写在备注导致主栏空白 |
| 装卸设备/叉车/尾板备用 | **精简一句**进备注 | 整段操作手册进备注 |
| 仓库下班时间/留卸货时间 | **精简一句**进备注 | 与时效字段重复堆叠 |

### 备注精简规则

- `remarks` 只保留**模板其他格子装不下**、且对派车/交付仍有用的操作要点。
- 控制在 **1～3 条短句**，建议总长 **≤ 80 字**；多用分号 `；` 分隔，不要段落作文。
- 可写：回单签字人与时限、现场叉车/尾板备用、仓库截单时间、需联系谁下楼开门（若主联系人栏已有姓名电话则备注只补“下楼开门”类动作）。
- 禁止：把用户整段口述/Agent 摘要原样粘贴；禁止重复已写入 `timeline` / `payment` / `handover` / `vehicleRequirement` / `cargo` 的内容。
- 信息很多时：**挑选会影响履约的 2～3 点**，其余可在对话中口头确认，不必进单。

**反例（禁止）**：把「提货方式…结算…保价…回单…装卸…装货提示…货物参数…下班时间…计划提货…车型…」整段作为备注。

**正例**：`回单：陈敏纸质签收当日带回；两端有叉车、尾板备用；莞仓 17:00 前留卸货时间`

## 单据自身字段

| 字段 | 说明 |
| --- | --- |
| 单据标题 | 默认“物流托运单”，有用户模板时保持原标题 |
| 单号 | 用户/系统已给出时使用；否则可建议可读编码，必须请用户确认后才定稿 |
| 制单日期 | 明确时区和日期；不将“明天”直接写入最终单据 |
| 单据状态 | 必填项未齐全时为“草稿·待确认”，齐全且已确认后为“已确认” |
| 签名/盖章 | 只预留位置，不伪造签名、印章或签收时间 |

## 字段置信度标注规则

- **高**：原文明确给出，格式合规
- **中**：原文有但表述模糊（如"大概两吨""明天左右"），需规范转换
- **低**：依赖识别（语音/手写）且结果存疑，或根据历史偏好补全，必须请客户确认

## 常见校验问题

- 电话与用户所在地区格式明显不符 → 标红追问原号码，不擅自补区号/国家码
- 地址只有城市名（如"发广州"）→ 追问区/镇 + 详细地址
- 货物只有品名没有数量重量 → 无法计价配载，必须追问
- "越快越好""尽快" → 转换为"加急"，并追问可接受的时效档位与期望日期
- 相对日期（明天/下周一）→ 按当前日期换算为绝对日期并显式请用户确认
