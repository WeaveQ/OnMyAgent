# 缺失信息追问话术模板

## 追问原则

1. **一次问清**：合并全部缺失/存疑项，一条消息问完，避免多次打扰。
2. **给选择题**：能提供选项的字段（时效档位、车型、结算方式）直接列出选项，客户回复序号即可。
3. **语气礼貌简短**：称呼 + 说明来意 + 编号列项 + 感谢，适合直接转发微信。
4. **标注优先级**：影响报价/派车的优先问，可后补的单独说明。
5. **只问当前阻塞项**：已确认字段不重复问；同一字段出现新旧冲突时同时列出两个值，请用户二选一。
6. **显式确认“无”**：包装、费用、保险、代收、特殊要求不能因未提及而默认没有；需要记录“无/不需要”时请用户明确确认。

## 自动合并顺序

每轮按以下顺序把所有阻塞项合并成一条消息，最多分为“客户信息”和“派车信息”两组，不逐字段发送多条消息：

1. 装卸地址与联系人电话
2. 货物名称、数量、包装、重量/体积
3. 绝对提货时间与到达时限
4. 结算方式与提货/交付方式
5. 冲突值、低置信度识别结果
6. 车型、费用、保价、回单和特殊运输要求
7. 仅最终版需要的车辆/司机信息

## 通用追问模板

```
X 总/X 经理，您好！您这票货（{货物}，{线路}）我已登记，还差几个信息想跟您确认一下：

1. {缺失项1}
2. {缺失项2}
3. {缺失项3}

确认后我马上安排录单发车，谢谢！
```

## 分场景话术

### 1. 缺联系方式
"麻烦发一下收货人的姓名和联系电话，方便司机到货前联系。"

### 2. 地址不完整
"收货地址目前只到'{市/区}'，麻烦补一下详细地址（区/镇 + 路名/门牌或地标），避免司机找不到耽误时效。"

### 3. 货物信息不全（无法计价）
"想跟您确认下这批{货物}的{件数/重量/体积}，我们好安排合适的车型并报准价。大概是：
1）{选项A}　2）{选项B}　3）其他（您直接说个数）"

### 4. 时效存疑（"越快越好"类）
"这票货给您按加急安排。想确认下：
1）今天发、明天到
2）明天发、后天到
您看哪个合适？"

### 5. 识别存疑（语音/手写）
"您发的{语音/单子照片}里有几个地方我怕听错/看错，跟您核对下：
1）收货人是'{识别结果}'对吗？
2）数量是'{识别结果}'吗？
没错的话回复'对的'就行。"

### 6. 历史偏好补全确认
"您上次发到{地址}、联系{人}{电话}，这次还按这个安排吗？有变化跟我说一声，没变回复'照旧'即可。"

### 7. 特殊要求确认
"这批{货物}需要{尾板车/冷藏车}对吧？另外{装卸}那边有叉车吗？我一次给您安排到位。"

### 8. 危险品提示
"您这批{货物}属于特殊物品，运输需要有对应资质。麻烦确认下货物具体品名{和类别/MSDS}，我帮您确认承运方案；如果不合规我们会如实告知，绝不冒险承运。"

## 高峰期批量跟进话术

"X 总，今天收到您 X 票订单，其中第 X 票（{简述}）还差{信息}，其他都已录好待发，麻烦您抽空确认下这票，马上给您安排。"

## 单据确认话术

客户必填已经齐全、尚未派车：

"这票客户信息已经补齐，我已按当前物流单汇总。请确认线路、双方信息、货物、提货/到达时间和结算方式是否无误。确认后我会同时生成 PDF 和 Excel 的‘待派车确认稿’；派车后补充车牌、驾驶证号、司机姓名和电话，再升级为‘最终版’。"

车辆与司机也已齐全：

"物流信息和派车信息已经补齐，请确认当前效果是否可以定稿。回复‘确认生成’后，我会同时生成 PDF 和双 Sheet Excel 最终版。"
