# 订单录入技能（Order Entry）

将碎片化订单素材转化为字段齐全、可落地的结构化运单的标准作业方法。

## 标准作业流程

1. **还原素材**：文字直接解析；语音先转写再按口语习惯理解（"老王""上次那个地址"等指代需结合上下文）；图片识别后对潦草字迹标注"识别存疑"。
2. **对照字段标准抽取**：按 `references/waybill-fields.md` 的必填/选填字段逐项抽取，每个字段标注置信度（高/中/低）。
3. **口语转规范表述**：按 `references/goods-and-requirements.md` 将口语化货物描述、时效与特殊要求转换为行业标准表述。
4. **完整性校验**：必填字段缺失、格式不合规（手机号位数、地址未细化到区/镇、货物信息无法计价）时标记为待补。
5. **生成追问话术**：按 `references/follow-up-scripts.md` 模板，把全部缺失/存疑项合并为一次礼貌追问，能给的选项直接列出。
6. **偏好匹配与确认**：老客户自动带出历史常用信息，全部标注"根据历史订单补全，请确认"。
7. **输出运单**：表格 + JSON 双格式，待确认项显式标记；批量场景生成台账逐单销项。

## 铁律

- 无依据的字段留空并追问，**禁止编造**手机号、地址、数量、价格。
- 追问一次问清，语气礼貌，给客户选择题而非问答题。
- 危化品、冷链、超限超重等特殊货物必须提示承运资质与包装要求。
