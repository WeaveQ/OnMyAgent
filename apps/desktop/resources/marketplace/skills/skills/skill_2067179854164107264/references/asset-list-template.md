# 数字资产清单模板与规则

用于 Step 7：数字资产清单生成。

## 定位

数字资产清单是剧本/分镜之后的**资产固化层**，用于后续 AI 视觉生成、视频生成、分镜深化、拍摄筹备、场景统筹、道具统筹、角色连续性管理。

它不是导演阐述，也不是制片预算表。它的任务是：把剧本中会反复出现、需要保持连续性、会被后续流程调用的元素固化成可检索清单。

## 必须包含的资产类别

### 1. 角色资产清单

每个主要/次要角色至少包含：

| 字段 | 说明 |
|:--|:--|
| 资产 ID | 稳定编号，如 `CHAR_001`；后续版本不得随意更改 |
| 角色名 | 剧本中的正式名称 |
| 别名/称呼 | 昵称、代称、化名、AI 生成时可识别的别称 |
| 版本状态 | 已锁定 / 未锁定 / 二稿待更新 / 需主创确认 |
| 角色级别 | 主角/核心配角/功能配角/群众 |
| 首次出场 | 集数、场号、场景 |
| 出场场次 | 汇总所有场号 |
| 年龄/身份 | 可从剧本确认的信息 |
| 外观连续性 | 发型、服装、标志物、伤痕、状态变化 |
| 关系绑定 | 与其他角色的核心关系 |
| 情绪/弧线状态 | 不同阶段的状态变化 |
| AI 生成注意 | 用于图像/视频生成时不能漂移的要点 |

### 2. 场景资产清单

每个反复出现或执行重要的场景至少包含：

| 字段 | 说明 |
|:--|:--|
| 资产 ID | 稳定编号，如 `SCN_001`；同一空间跨场次复用同一 ID |
| 场景名 | 如“容青可公寓”“苏家客厅” |
| 别名/空间组 | 同一空间的不同叫法、子空间或所属场景组 |
| 版本状态 | 已锁定 / 未锁定 / 二稿待更新 / 需主创确认 |
| 场景级别 | 主场景/次主场景/一次性场景 |
| 出现场次 | 集数与场号 |
| 空间功能 | 叙事功能和人物关系功能 |
| 视觉关键词 | 色彩、光线、材质、空间气质 |
| 关键区域 | 门、窗、沙发、书桌、楼梯等 |
| 连续性要求 | 布置变化、时间变化、损坏/血迹/道具位置等 |
| AI 生成注意 | 保持空间一致性的提示 |

### 3. 道具/物件资产清单

每个承担叙事功能的物件至少包含：

| 字段 | 说明 |
|:--|:--|
| 资产 ID | 稳定编号，如 `PROP_001`；同一道具跨场次复用同一 ID |
| 道具名 | 猫照片、薄荷糖、白熊服等 |
| 别名/道具组 | 其他叫法、同组道具、可合并管理的物件组 |
| 版本状态 | 已锁定 / 未锁定 / 二稿待更新 / 需主创确认 |
| 道具级别 | 核心母题/关键道具/生活道具 |
| 首次出现 | 集数、场号 |
| 出现场次 | 汇总所有场号 |
| 叙事功能 | 推动剧情、承载记忆、关系转折、情绪回收等 |
| 外观/状态 | 颜色、材质、数量、破损、使用痕迹 |
| 连续性变化 | 从首次出现到回收的变化 |
| AI 生成注意 | 生成图像/视频时不可漂移的特征 |

### 4. 母题/意象资产清单

适用于反复出现但不一定是实体道具的元素：雨、薄荷气息、窗户、门、铁栅栏、梦境沼泽等。

| 字段 | 说明 |
|:--|:--|
| 资产 ID | 稳定编号，如 `MOTIF_001`；同一母题跨场次复用同一 ID |
| 母题名 | 意象或重复视觉元素 |
| 别名/关联资产 | 相关道具、场景、声音或动作 |
| 版本状态 | 已锁定 / 未锁定 / 二稿待更新 / 需主创确认 |
| 类型 | 视觉/声音/气味/空间/动作 |
| 出现场次 | 汇总场号 |
| 主题功能 | 它承担的主题含义 |
| 变化路径 | 如何从前期到后期变形 |
| 执行方式 | 剧本、分镜、AI 生成、声音设计中的表现方式 |
| 禁止误用 | 不能变成什么，例如不能把创伤母题拍成浪漫装饰 |

### 5. 服装/妆发/状态连续性清单

尤其适用于跨时间线、受伤、剪发、换装、年龄变化、职业变化的项目。

| 字段 | 说明 |
|:--|:--|
| 角色名 | 关联角色 |
| 阶段 | 集数/剧情阶段 |
| 服装/妆发 | 可执行描述 |
| 身体状态 | 生病、受伤、疲惫、血迹、纱布等 |
| 连续性场次 | 需要保持一致的场号 |
| 变化原因 | 剧情中为什么变化 |

## 资产稳定偏好

生成数字资产清单时，默认采用以下稳定偏好：

1. **稳定 ID 优先**：主要角色、复用场景、关键道具、核心母题必须分配资产 ID。推荐前缀：`CHAR_` 角色、`SCN_` 场景、`PROP_` 道具、`MOTIF_` 母题、`COSTUME_` 服装状态。
2. **主名称不漂移**：资产一旦确认主名称，后续版本不得随意改名；其他叫法写入“别名/称呼/备注”。
3. **版本状态明确**：区分已锁定、未锁定、二稿待更新、需主创确认，避免把草表当成最终设定。
4. **来源场次可追溯**：重要资产必须写明首次出现和主要出现场次，便于剧本改稿后更新。
5. **连续性锁定**：脸、年龄、发型、服装、伤痕、道具外观、空间布局、母题用法等不能在后续 AI 生成中漂移。
6. **变更不重建**：剧本进入二稿/三稿时，优先保留原资产 ID，生成变更记录；不要整表重命名或重排导致后续资产失效。
7. **未知不硬编**：原文或剧本没有确认的信息，用“待定/需主创确认”，不得为了表格完整擅自定死。

## 生成原则

1. **基于已确认剧本/分镜**：不得新增剧情、角色、场景或道具。
2. **优先固化高复用资产**：反复出现、承担母题、影响连续性的资产优先。
3. **为 AI 后续流程服务**：字段必须能被用于 AI 图像生成、视频生成、分镜扩展、角色一致性控制。
4. **区分叙事功能与视觉特征**：既要说明“为什么重要”，也要说明“长什么样/怎么保持一致”。
5. **标出连续性风险**：剪发、受伤、房间被打扫、照片位置、时间线切换等必须明确。
6. **避免泛泛描述**：不要只写“漂亮”“悲伤”“高级”，要写可执行、可检索、可复用的描述。

## 输出建议

长项目建议分批：

1. 先生成全项目资产总表；
2. 再按类别展开：角色 / 场景 / 道具 / 母题 / 服装妆发；
3. 必要时为 AI 生成另存“角色一致性提示包”“场景一致性提示包”。

若剧本已进入二稿/三稿，或用户提供了新版分镜，必须追加“资产变更记录”：

| 字段 | 说明 |
|:--|:--|
| 变更类型 | 新增 / 删除 / 合并 / 拆分 / 改名 / 外观变化 / 场次变化 / 连续性风险 |
| 资产 ID | 沿用原 ID；新增资产再分配新 ID |
| 资产名 | 主名称 |
| 变更依据 | 来自哪一稿剧本、哪一场、哪一版分镜 |
| 变更内容 | 具体改了什么 |
| 后续影响 | 对 AI 生成、分镜、拍摄、道具/服装连续性的影响 |

## 文件名

默认文件名：

`项目名_数字资产清单.md`

如用户偏传统影视流程，也可命名为：

`项目名_场景道具角色清单.md`
