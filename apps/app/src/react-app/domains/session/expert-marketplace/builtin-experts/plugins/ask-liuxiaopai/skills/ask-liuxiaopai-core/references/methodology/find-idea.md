# 找 idea 方法论：幸福路径 vs 功利路径

> **何时加载**：用户问题包含 不知道做啥 / 没方向 / 找 idea / 没想法 / 该做什么

idea 不是"想"出来的，是**发现**出来的。两条并行路径——按用户背景选。

---

## 路径 A：幸福路径（D3 - 收集抱怨）

**适合谁**：已经在某个行业 / 角色深度浸泡过的人（前从业者、长期用户、深度爱好者）

**核心**：

> **真正有价值的需求，不藏在用户说"我想要什么"里——藏在他们已经受不了、反复吐槽、甚至公开开骂的地方。**

### 收集抱怨 4 步法

1. **去抱怨密度高的地方翻**：差评区、知乎吐槽、微博骂楼、竞品 1 星评论、行业群、Reddit 子版
2. **记录原话**：不要总结，逐字抄。语音笔记 + 日复盘
3. **筛 3 个问题**：
   - 这个抱怨**重复出现**了吗？（一次性吐槽 vs 系统性痛点）
   - 这群抱怨者是**同一类人**吗？（同一画像才能砍出 wedge）
   - 市场已经有解决方案，但他们**还在抱怨**吗？（说明现有方案不够好）
4. **找 1 个真人聊**：不要群发，找一个具体的抱怨者，深度访谈

**优势**：自己最懂、竞争对手少、用户忠诚度高

**风险**：市场规模可能小、周期长

---

## 路径 B：功利路径（D4 - 五大方法）

**适合谁**：需要快速正反馈的新手、想抓窗口期的人、缺乏行业积累的转型者

**核心**：寻找**供需失衡**——需求已存在 + 供给还不够好 = 机会窗口

### 5 大功利方法

1. **搜索词**：Google Trends + 百度指数看上升曲线，找需求量大但供给少的词
2. **新闻窗**：政策/事件/合规变化创造的临时需求（"双减""碳中和""个保法"）
3. **新技术**：新模型/新平台/新协议（GPT 出来时的 wrapper 套利）
4. **暗影复刻**：海外有但国内没有 / 国内有但海外没有 / B 端验证过搬到 C 端
5. **服务产品化**：你身边有人靠手工赚钱 → 想想能否产品化

### 机会表（10 选 1）

把每个发现的机会都填进表，每月跑一次：

| 字段 | 怎么填 |
|---|---|
| 机会名 | 一句话 |
| 来源 | 搜索词/新闻/新技术/复刻/服务化 |
| 目标用户 | 具体的人，不是人群 |
| 现状 | 用户现在怎么凑合 |
| 最差体验 | 用户对现有方案最不爽什么 |
| 最小产品 | 一个人能在 2 周内交付的版本 |
| 技术路径 | 大致用什么栈 |
| 推广渠道 | 第一批 10 个用户从哪 |
| 收费方式 | 一次性 vs 订阅 / 多少钱 |
| 风险 | 最大不确定性是什么 |

**舍九取一**：每 10 个 idea 只做 1 个。其他 9 个不是不好，是没时间。

**优势**：需求已验证、时间短、现金流快

**风险**：容易过时、进入壁垒低

---

## 两条路径的选择

| 信号 | 选哪条 |
|---|---|
| 你已经在某个行业泡了 5+ 年 | 路径 A |
| 你是转型者，想快速找到新方向 | 路径 B |
| 你是 AI 开发者，懂技术但不懂行业 | 路径 B（用搜索词/新技术发现机会） |
| 你是内容创作者，懂分发不懂技术 | 路径 A（用粉丝抱怨找 wedge） |
| 你想做长期复利的事 | 路径 A |
| 你急着拿到第一桶金 | 路径 B |

**两条路径不互斥**——好 idea 往往是 A + B 双重验证（自己的痛 + 市场已有需求曲线）。

---

## 危险信号

- "我想做一个能解决很多人问题的工具" → 假设需求被发明
- "我觉得这个方向很有前景" → 没看真实抱怨
- "我朋友说他可能会用" → 兴趣不是需求
- "市场调研报告说这是 X 亿市场" → 报告不是用户

---

## 收尾必做

不要只给方法。让用户**今晚就开始**：

- 路径 A 用户 → "今晚去 [具体平台] 翻 30 条抱怨，记原话，明天我们来看"
- 路径 B 用户 → "今晚把你最近一周看到的 3 个新词列出来，搜一下指数曲线"
