## 二、按能力看功能分组

### A. 查询类功能

| 功能 | 解决什么问题 | 常见问法 | 典型入口 |
|---|---|---|---|
| 游戏时长查询 | 看孩子/本人玩了多久 | 今天玩了多久、近 7 天玩了哪些游戏 | 游戏管理页、AI 助手、周报/记录页 |
| 充值记录查询 | 看消费金额与订单 | 本月充了多少、昨天有没有充值 | 游戏管理页、AI 助手 |
| 查询身份证名下账号 | 排查孩子是否冒用家长身份/账号 | 我名下有几个号、孩子是不是用我的号玩 | AI 助手 / 需强身份核验 |
| 游戏周报 | 看每周时长、消费趋势 | 最近一周玩得多不多 | 我的页 → 游戏周报 |
| 小程序记录查询 | 看微信小程序使用情况 | 最近用了哪些小程序 | 游戏管理 → 小程序 |

### B. 直接管控类功能

| 功能 | 解决什么问题 | 更适合的场景 | 备注 |
|---|---|---|---|
| 游戏时间限制 | 控制每天可玩时长或时段 | 日常稳定管理、想减少时长 | 对未成年规则不能放宽 |
| 游戏充值限制 | 控制单次/月度充值 | 担心冲动消费、误充 | 对未成年规则不能放宽 |
| 游戏禁玩 / 一键禁玩 | 禁止某游戏或直接强限制 | 高风险、立即止损 | 要区分“全游戏限时”与“单游戏禁玩” |
| 临时锁 / 游戏临时开关 / 临时放行 | 临时允许玩 X 分钟或临时覆盖禁令 | 临时借手机、家长在旁陪玩 | 资料显示优先级可高于既有禁玩/限时 |
| 人脸识别管控 / 自助下发人脸识别 | 防止孩子冒用家长账号 | 孩子用家长账号玩 | 需家长身份核验 |
| 绿色手机上报 | 把某微信账号纳入未成年规则 | 孩子使用家长或亲友微信账号 | 更偏“孩子用他人账号”场景 |
| 守护锁 | 防止孩子偷偷改设置/解绑 | 家长担心孩子拿自己手机改方案 | 开关都需身份验证 |
| 微信小程序管理 | 管总时长或单款小程序 | 不只是游戏，也要控小程序娱乐 | 包括总时长/单款管理/禁用 |

## 三、优先使用的场景映射

### 1. 孩子用自己的账号玩
优先链路：

1. 家庭管理绑定孩子账号
2. 如要管控，再做孩子身份验证
3. 再进入时间限制 / 充值限制 / 游戏禁玩 / 小程序管理
4. 需要看数据时，可查时长、充值、登录、周报

### 2. 孩子用家长账号玩
优先链路：

1. 先确认是不是家长当前登录账号
2. 视目标使用临时锁/临时放行、人脸识别管控、绿色手机上报、守护锁
3. 如果想解释“为什么还能玩”，还要结合当前生效规则解释

### 3. 孩子用别人账号玩
优先链路：

1. 如果知道是谁的账号，可优先考虑绿色手机上报或让对方配合
2. 如果不知道账号，先说明边界：没有明确账号很难直接管控
3. 可以用身份证名下账号查询、疑似小号排查、家长服务助手作为补充方案

## 四、回答时的页面口径

- **不要把旧入口和新入口说成冲突**。正确表述应是：产品存在老版/新版页面表现，核心能力一致，但入口展示方式不同。
- **优先说“功能层级”而不是死记页面坐标**。例如先说“先到游戏管理里的时间限制”，再补“如果你看到新版 AI 页，也可以先从 AI 助手或底部管控工具进入”。
- **对新版产品要强调 AI 是统一入口，不等于替代所有工具**。AI 的价值是意图识别、补齐参数、生成方案并帮助一键执行。
- **对全功能聚合页要区分主线与补充**。主线仍是游戏管理与管控，社区、课程、资讯、亲子沟通属于辅助内容。

## 五、需要谨慎回答的点

- “屏蔽陌生人”“一键联系家长”等功能，在你提供的资料里只有页面露出，没有完整规则说明；回答时可说它属于安全/提醒类能力，但不要编造细节。
- “家长柜台”和“家长服务助手”都偏复杂问题升级处理；如果问题涉及异常、申诉、材料提交、人工审核，可优先引到这两个入口。
- 如果用户问的是健康系统、实名、人脸、举报、热线等问题，优先联动 `faq-routing.md` 回答。
