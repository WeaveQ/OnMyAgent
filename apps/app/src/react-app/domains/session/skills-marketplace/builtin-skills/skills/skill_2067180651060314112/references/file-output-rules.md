# File Output Rules

本 Skill 的正式输出必须保存为 Markdown 文件，不应只贴在会话中。

---

## 必须写入文件的输出

以下内容一律保存为 `.md`：

- 快速诊断
- 完整开发读本
- 演员读本
- 编剧改编读本
- 导演视听读本
- 制片摘要
- AI 剧本化提示包
- 多模式组合输出
- 阶段性开发诊断

会话中只返回：

1. 输出完成说明；
2. 生成文件列表；
3. 每个文件 1-3 句摘要；
4. Markdown 文件链接。

---

## 保存位置原则

不要硬编码某个应用或某台设备的绝对路径。优先使用当前 Agent / 当前运行环境的默认工作区、项目目录或用户指定目录。

推荐顺序：

1. **用户指定目录**：如果用户明确给出路径、项目目录或挂载目录，按用户指定位置保存。
2. **当前项目目录**：如果当前会话已有项目文件夹或输入文件所在目录适合写入，保存到该项目目录下的子目录，如 `narrative-to-screen-reader/` 或 `screen-reader-output/`。
3. **Agent 默认工作区**：如果没有项目目录，使用当前 Agent 环境的默认可写工作区。
4. **共享/持久目录**：如果用户需要跨会话复用、交付、开源或长期保存，使用当前环境提供的共享/持久目录。
5. **无法判断时**：创建一个清晰命名的输出文件夹，如 `narrative-to-screen-reader-output/`，并在回复中说明保存位置。

在 Minis 环境中可使用 `/var/minis/workspace/` 或 `/var/minis/shared/`；但这只是本环境示例，Skill 不应假设所有用户都使用相同路径。

---

## 文件命名规则

格式：

`<故事名>_<输出模式>.md`

示例：

- `同一条河_快速诊断.md`
- `同一条河_影视化开发读本.md`
- `同一条河_演员读本.md`
- `同一条河_导演视听读本.md`
- `同一条河_制片摘要.md`
- `同一条河_AI剧本化提示包.md`

若故事名未知，用：

`未命名故事_快速诊断.md`

若同名文件已存在，可追加版本号：

`同一条河_演员读本_v2.md`

---

## 会话回复格式

生成文件后，会话回复应简洁：

```markdown
已生成：

- [同一条河_快速诊断.md](minis://...)
- [同一条河_演员读本.md](minis://...)

简要判断：
这个故事适合进入完整开发读本，人物与物件链成立。下一步建议生成 AI 剧本化提示包。
```

不要把完整正文再次粘贴到会话里，除非用户明确要求“直接贴出来”。

---

## 多文件输出

当用户要求多个模式时，分别生成多个 `.md` 文件，而不是塞进一个超长文件。

可以额外生成一个总目录：

`<故事名>_输出目录.md`

目录内容包括：文件清单、用途说明、推荐阅读顺序。
