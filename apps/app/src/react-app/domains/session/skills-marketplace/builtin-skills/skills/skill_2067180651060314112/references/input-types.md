# Input Types

先判断文本类型，再决定分析策略。不要所有文本套同一格式。

---

## 1. 长会话角色扮演记录

特征：多轮对话、用户/助手角色交替、信息量大且分散、关系自然生长。

策略：

- 先按轮次或时间线提取主线。
- 合并重复推进，保留关键情绪节点。
- 特别关注：已读不回、时间戳、停顿、消息格式、用户动作、环境描写、NPC 反应。
- 抓“自然长出来”的物件链，不只抓显性台词。
- 判断终局是否来自角色人格和结构，而非作者硬拧。

适合输出：完整开发读本、演员读本、AI 剧本化提示包。

---

## 2. 短篇小说

特征：文本凝练，文学意象集中，可能有大量内心描写。

策略：

- 分析叙述视角：第一人称、第三人称、限制视角等。
- 找可影视化场景，避免只分析抽象主题。
- 把内心独白转化为动作、空间、物件、声音的可能性。
- 判断哪些文学句子必须保留为旁白，哪些应转为镜头。

适合输出：编剧改编读本、导演视听读本、完整开发读本。

---

## 3. 中长篇小说片段

特征：上下文可能不完整，人物关系可能已有前史。

策略：

- 明确“基于当前片段”的分析边界。
- 不擅自补完前史。
- 标注待补信息：人物关系、前置事件、终局方向、世界观规则。
- 输出片段影视化价值和接续问题。

适合输出：快速诊断、编剧改编读本、待补清单。

---

## 4. 故事大纲 / 梗概

特征：有事件，少细节；人物动机、动作、物件可能不足。

策略：

- 不强行做细致演员读本，除非人物材料足够。
- 重点做开发诊断：结构、动机、冲突、场景缺口、视听承载不足。
- 建议关键场景、关键物件、关系节点和情绪节拍。

适合输出：快速诊断、编剧改编读本、制片摘要。

---

## 5. 剧本初稿

特征：已有场景、对白、动作，可能需要深层分析和修订建议。

策略：

- 分析每场戏的功能：推进情节、揭示人物、制造转折、建立母题。
- 检查台词是否只说表层，是否有潜台词。
- 判断动作是否可演、场景是否可拍。
- 给出保留/删减/合并建议。

适合输出：编剧改编读本、演员读本、导演视听读本。

---

## 6. 互动叙事 / 游戏文本

特征：存在玩家选择、分支、好感度或多结局。

策略：

- 识别主路径和关键分支节点。
- 分析玩家选择对人物关系的意义。
- 判断最适合影视化的主路径。
- 不试图保留所有分支，优先确定影视版主线。

适合输出：编剧改编读本、项目摘要、AI 剧本化提示包。
