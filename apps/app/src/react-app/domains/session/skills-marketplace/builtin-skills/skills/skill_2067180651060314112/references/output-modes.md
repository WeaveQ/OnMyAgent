# Output Modes

根据用户目标选择输出模式。若用户未指定，先输出“快速诊断 + 推荐输出模式”。

---

## A. 完整开发读本

适用：重要文本、准备进入剧本化、需要完整开发材料。

如果文本已具开发价值，但人物厚度不均、部分角色可演性不足，可对个别角色启用“补强说明”或附补强型读本。若配角具有结构作用，可加入“重要配角功能表”，不为每个配角单独写完整读本。

完整开发读本是**母文档**。其他岗位模块应尽量基于它拆分，不要各说各话。

建议结构：

1. **总体判断**：一句话核心 + 类型/情感结构 + 终局自然性 + 最关键总判断。
2. **故事核心命题**：这故事到底在讲什么；它不只是表层在讲什么。
3. **人物读本**：主角、关系主轴对象，必要时重要关系对象。
4. **关系读本**：关系如何开始、推进、变化，其独特性在哪里。
5. **逐幕结构分析**：每幕/阶段为什么存在；删掉会塌什么。
6. **关键物件链**：首次功能 / 后续意义 / 终局回收 / 影视化处理。
7. **空间链与视听母题**：空间功能、声音、光线、颜色、气味、留空。
8. **重要配角功能表**：功能类型、与主角关系、作用、是否必须保留。
9. **改编风险**：最容易把它做坏的方式、最容易误判的类型。
10. **第三步剧本化建议**：必须保留、可合并、可删减、需转译、禁止方向。

详见 `full-development-reader.md`。

## B. 演员读本

适用：演员准备角色或表演指导。

重点不是解释角色“是什么人”，而是回答：
- 这个人怎么活着；
- 怎么说话；
- 怎么站、坐、看、停；
- 一遇到对手戏对象会变成什么样；
- 最容易演错成什么。

可分为两种：

- **标准演员读本**：原文细节足够，人物已有明确动作、身体逻辑和情绪显影。
- **补强型演员读本**：故事成立，但人物描写较薄，需基于行为逻辑、说话节奏、关键选择和物件/空间习惯提炼出可演结构。

建议结构：

1. 角色核心命题
2. 最不想让别人看见的东西
3. 外在行为习惯
4. 说话节奏与语言禁区
5. 情绪表达方式
6. 防御方式
7. 如何靠近 / 如何退
8. 最重要的表演点
9. 不能演错的地方
10. 一句话表演方向

补强型演员读本可额外附：
- 已知人物事实
- 补强依据
- 不可过度补强的边界

详见 `actor-reader.md`。

## C. 编剧改编读本

适用：小说/长会话/大纲进入正式剧本化前；尤其适合群像、多主角、长篇、强设定、骨架强但场景不足的文本。

它的核心任务不是再分析“故事好不好”，而是回答：
- 该从哪里切进去改；
- 哪些不能动；
- 哪些要扩成戏；
- 哪些可以删 / 并 / 压；
- 最危险的误改方式是什么。

建议结构：

1. 改编入口判断
2. 开发形态判断（长片 / 剧集 / 短片 / 单元集等）
3. 结构骨架
4. 必须保留项（场景 / 物件 / 台词 / 关系质地）
5. 可删 / 可并 / 可压缩项
6. 必须扩写成戏的部分
7. 内心独白 / 抽象命题转译方案
8. 重要配角保留与合并建议
9. 误改风险
10. 下一步执行建议（主建议 1 条 + 备选 1 条)

详见 `screenwriter-adaptation-reader.md`。

## D. 导演视听读本

适用：导演、摄影、美术、声音、分镜前期。

它的核心任务不是做分镜表，而是判断：
- 这个故事真正该拍的是什么；
- 哪些必须交给镜头和声音；
- 哪些不能说满；
- 最容易被怎么拍坏。

如果文本人物成立但场景承载不足，本模式应指出：哪些情绪目前只能靠文字说明，哪些需要补一场戏、一个空间或一个物件，才能真正视听化。

建议结构：

1. 总体视听判断
2. 视觉母题
3. 声音母题
4. 主要空间及情绪属性
5. 视听执行重点
6. 节奏与留空
7. 最危险的误拍方式
8. 一句话导演提醒

详见 `director-visual-reader.md`。

## E. 制片/项目开发摘要

适用：立项、给老板/制片/平台方快速判断。

它的任务不是再讲一遍故事，而是回答：
- 这个项目到底是什么；
- 靠什么吸引观众；
- 最值钱的东西是什么；
- 最适合做成什么形态；
- 最容易死在哪。

建议控制在 800-1500 字。

建议结构：

1. 一句话项目定位
2. 类型与情绪卖点
3. 当前最值钱的东西
4. 核心受众
5. 改编优势
6. 适合的项目形态
7. 制作规模（粗估）
8. 项目风险（创作风险 / 制作风险）
9. 是否建议推进 + 下一步

避免：长篇心理分析、过细表演指导。涉及预算、周期、片长、集数、演员数量等数字时，必须写明“粗估/假设/需正式制片预算复核”；不要把开发阶段估算写成确定制片方案。

详见 `producer-brief.md`。

## F. AI 剧本化提示包

适用：下一阶段让 AI 生成正式剧本。

建议结构：

1. 故事核心不可改
2. 角色行为规则
3. 必须保留场景
4. 必须保留台词
5. 必须保留物件
6. 重要配角必须保留项 / 可合并项
7. 禁止改写方向
8. 情绪节奏
9. 视听提示
10. 场景列表
11. 可直接给 AI 的剧本化 Prompt

详见 `ai-screenplay-prompt-pack.md`。

## G. 快速诊断

适用：初步判断、用户未指定输出模式、文本较长前的预分析，尤其适合故事尚不成熟或用户不确定下一步时。

控制在 1000-2000 字。

必须包含：

1. 诊断等级：A 可直接进入完整读本 / B 可做读本但需补局部信息 / C 暂不适合完整读本，适合开发诊断 / D 需重构 / D0 不可开发。
2. 故事核心是否清楚。
3. 人物是否成立：欲望、恐惧、防御、行动。
4. 最强内容：场景、台词、物件或关系节点。
5. 最大问题：明确指出 1-3 个最影响影视化的问题。
6. 是否适合生成读本：适合哪种、不适合哪种、为什么。
7. 下一步建议：给出可执行补写/改写清单；必要时说明应优先补“人物厚度、身体逻辑、场景承载、物件线”中的哪一项。

详见 `quick-diagnosis-guide.md`。
