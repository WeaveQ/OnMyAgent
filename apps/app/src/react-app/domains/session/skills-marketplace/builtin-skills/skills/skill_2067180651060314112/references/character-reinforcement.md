# Character Reinforcement

用于处理一切**故事已成立、具备开发价值，但角色厚度、可演性或身体逻辑不足**的情况。

它不只服务长会话，也适用于：

- 长会话角色扮演记录
- 轻小说 / 网文短篇
- 中短篇小说
- 故事大纲 / 梗概
- 剧本初稿中功能性过强的人物

本机制的目标不是“把角色写丰满”，而是：

> **在不新增剧情事实的前提下，让一个本来偏薄的人物，达到演员能演、导演能拍、编剧能继续改的最低厚度。**

---

## 一、什么时候触发补强

满足以下任一情况，可启用角色补强机制：

1. 角色在故事结构里很重要，但目前更像功能支点而不是活人。
2. 角色推动了关键剧情，但几乎没有静态描写、身体逻辑或情绪显影。
3. 角色心理动机成立，但演员读本难以直接生成。
4. 核心对象很厚，另一方人物相对偏薄。
5. 轻小说/短篇/大纲里的人物标签成立，但不够可演。
6. 剧本初稿中某角色对剧情重要，但场景里几乎只有信息功能。
7. 故事已经通过快速诊断（A/B），但人物层面仍有“可开发但不够稳”的情况。

---

## 二、补强不是编造

### 禁止做的事

- 新增原文没有的童年创伤、家庭矛盾、前任经历、人格疾病等设定；
- 为了“更完整”而补写新的剧情事实；
- 把作者/用户本人的现实经验直接嫁接给角色；
- 把推断写成原文明确事实；
- 用俗套人格模板（高冷、腹黑、病娇、圣母、霸总等）覆盖角色原有逻辑。

### 可以做的事

只能从以下材料中提炼与补强：

1. **反复出现的行动方式**
   - 他/她总是怎么做决定？
   - 真正关键时刻怎么选？

2. **稳定的说话节奏**
   - 先讲逻辑、先讲感觉、先转移话题，还是先让步？
   - 句子长短、术语密度、反问频率、是否善于安抚等。

3. **面对压力时的第一反应**
   - 退回系统语言、沉默、讲笑话、继续做事、切换到现实事务等。

4. **给入口 / 接入口的方式**
   - 是主动提供位置、发消息、继续出现、留一份东西，还是以承担来靠近？

5. **别人如何看见他/她**
   - 对手戏角色、旁人、环境对这个人物的判断。

6. **空间与物件习惯**
   - 桌面、车、包、手机、工作习惯、姿势、道具使用等。

7. **未说出口的东西如何泄露**
   - 手、停顿、回避、未发生的动作、保留物等。

## 三、补强的三种方向

### A. 人物厚度补强

适用：角色有功能、有命题位置，但不够像活人。

补强内容：
- 欲望
- 恐惧
- 防御方式
- 行动逻辑
- 说话方式
- 不可演错之处

### B. 身体逻辑补强

适用：角色心理成立，但演员拿到手不知怎么演。

补强内容：
- 站姿 / 坐姿 / 走路
- 看人方式
- 手部动作
- 停顿节奏
- 面对压力时身体怎么反应
- 与空间和物件的关系

### C. 场景承载补强

适用：角色有了，但缺少“戏”来承载人物。

补强内容：
- 暴露性格的场景
- 做选择的场景
- 与物件发生关系的场景
- 私人空间 / 工作空间 / 边界空间

---

## 四、补强型演员读本格式

当某角色原文偏薄，但值得开发时，演员读本可采用以下结构：

### 1. 已知人物事实

只写原文明确给出的职业、身份、立场、已发生行为。

### 2. 可稳定提炼的行为逻辑

这是补强核心。

写清楚：
- 他/她处理问题的方式；
- 情绪一上来先怎么防；
- 靠近方式；
- 最像哪种生存方式的人。

### 3. 演员可用的身体逻辑

从行为模式反推：
- 说话前会不会看屏幕/资料/杯子；
- 会不会先停一下；
- 身体是稳、乱、收、压、硬还是松；
- 压力下动作变快还是变少。

### 4. 可演的弧线

不是剧情梗概，而是演员能演出的变化：
- 从什么状态到什么状态；
- 通过语言、动作、选择还是沉默完成变化；
- 变化的最小表演单位是什么。

### 5. 不可过度补强的边界

明确哪些是原文事实，哪些是基于原文的提纯，哪些绝对不能写成既定设定。

## 五、对不同文本类型的适用说明

### 长会话

常见情况：核心对象厚，参与者角色薄。

处理：
- 先做 `dialogue-text-conversion.md`，把轮次转成戏剧单元；
- 再判断较薄的一方是否具有稳定行为逻辑；
- 如有，启用补强型读本。

### 轻小说 / 网文短篇

常见情况：剧情流畅，人物标签成立，但身体逻辑和场景承载不足。

处理：
- 不直接否定角色；
- 重点补“可演性”和“可拍性”。

### 大纲 / 梗概

常见情况：命题成立，人物模糊，场景缺失。

处理：
- 补强以“开发建议”为主，不直接生成厚演员读本；
- 先建议补三类场景，再进演员读本。

### 剧本初稿

常见情况：某角色功能清楚，但台词和动作都过于服务剧情。

处理：
- 补强人物厚度与身体逻辑；
- 让角色从“说台词的人”变成“在场的人”。

## 六、判断一句话有没有补强价值

优先问：

> 这句话不是“他/她怎么说”，而是“为什么总这样说”？

例如：
- 总先讲系统逻辑 → 可能是用系统语言做防御；
- 总给空间不逼答复 → 可能是尊重边界，也可能是怕伤人；
- 总把关系讲成预算、流程、项目 → 可能是他/她组织世界的方式；
- 总先调侃后收住 → 可能是热防御；
- 总把问题拉回专业细节 → 可能是在保护尊严。

这些都可以转化为可演的人物结构。

---

## 七、输出时的措辞要求

补强时要明确区分：

- **原文事实**：明确写出
- **基于行为逻辑的提纯**：可以写，但要避免过度肯定
- **开发建议**：明确是为剧本化服务的建议

推荐句式：
- “从原文稳定行为看，他更像……”
- “演员可按这个方向处理……”
- “这不是原文明说，而是基于其连续选择可提炼出的逻辑……”

避免句式：
- “他童年一定……”
- “她其实从小……”
- “他一直都是因为……”（若原文无证据）

---

## 八、最终提醒

角色补强机制的目标不是“让每个人都很丰满”，而是：

> **当故事已经成立、值得开发时，不要因为人物描写略薄，就让它停在不能拍、不能演、不能继续改的阶段。**

补强不是扩写设定，而是让角色站起来。
