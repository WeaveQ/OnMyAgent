# 核心场景清单

本文件用于在回答前快速完成“场景判断 → 功能选配 → 建议表达”。本版已结合功能清单、AI 产品说明书、新旧产品页和 FAQ 资料校准。

## 使用方法

1. 先判断用户最关心的是“理解、查询、判断、选配、操作、校准、服务分流”中的哪一种。
2. 再判断孩子在用的是自己的账号、家长账号还是其他人的账号。
3. 先给 P0 主方案，再给 P1 补充方案。
4. 写入类操作必须提醒二次确认；高风险 FAQ 必须分流到官方入口。

## 场景 1：家长不知道孩子为什么还能玩

### 典型提问
- 为什么孩子还能玩腾讯游戏？
- 明明我以为有限制，为什么还是能登录？
- 国家不是有限制吗，为什么白天还能玩？

### 场景本质
家长首先缺的不是某一个功能，而是对“账号来源、实名状态、健康系统规则、成长守护当前是否已生效”的理解。

### 回答重点
- 先判断孩子是在用自己的账号、家长账号还是别人的账号。
- 再解释“为什么还能玩”通常要从实名年龄、是否已完成绑定和身份验证、当前是否存在临时放行/临时锁覆盖、账号是否不在管控范围内几个角度排查。

### 推荐方向
- **P0**：先查当前账号和规则状态，再进入对应的时长、消费、安全能力配置。
- **P1**：如果怀疑孩子换号或冒用家长账号，再叠加身份证名下账号查询、绿色手机上报、人脸识别管控、家长服务助手等方案。

## 场景 2：家长想临时让孩子玩一阵，但要可控

### 典型提问
- 我把手机借给孩子玩一会儿，怎么控制比较稳？
- 能不能只让他玩半小时，不要乱充钱？
- 不想完全禁玩，只想临时放开一下。

### 场景本质
核心目标不是“禁玩”，而是“临时放行但风险可控”。

### 推荐方向
- **P0**：优先推荐临时锁/游戏临时开关 + 消费限制 + 必要的安全能力。
- **P1**：如果孩子使用的是家长账号，可进一步建议开启人脸识别管控或守护锁，避免放行结束后被继续冒用。

### 表达原则
- 强调“先定目标，再定时长，再补风险控制”。
- 要提醒临时放行可能覆盖原有限制，到期后应说明是否恢复原规则。

---

## 场景 3：家长要做日常稳定管理

### 典型提问
- 我想长期控制孩子玩游戏的时间和消费，怎么配比较合适？
- 不是要完全禁玩，只想日常管住。

### 场景本质
目标是建立可持续的家庭管控方案，而不是一次性止损。

### 推荐方向
- **P0**：按“时间限制 + 充值限制”两条主线配置基础能力。
- **P1**：视风险情况叠加守护锁、登录提醒、周报、微信小程序管理或指定游戏禁玩。

### 设置建议
- 给出“平日/周末、低龄/高龄、一般管理/加强管理”的差异化建议。
- 说明功能搭配逻辑，不要只列功能名。

## 场景 4：孩子已出现高风险行为，需要强管控

### 典型提问
- 孩子偷玩、夜间玩、乱充钱，怎么办？
- 我现在想先强行管住，再慢慢调整。

### 场景本质
这是高风险、低容错场景，优先级是止损和封堵风险。

### 推荐方向
- **P0**：优先推荐强限制能力，如游戏禁玩、一键强限制、充值限制、守护锁、人脸识别管控。
- **P1**：在止损后，再补充更长期的时长管理、周报观察和亲子沟通建议。

### 表达原则
- 先给“立刻怎么做”，再讲“后续怎么恢复到更平衡的管理”。
- 如果孩子在用其他账号，要明确说明当前账号限制未必能覆盖全部情况。

## 场景 5：家长不知道时间该限制多少才合适

### 典型提问
- 时间限制设多少比较合理？
- 管太严怕反弹，太松又担心沉迷，怎么办？

### 场景本质
用户要的不是一个数字，而是“一个有依据的建议范围”。

### 判断维度
- 年龄段 / 学段
- 平日还是节假日
- 当前风险等级：普通娱乐、已影响作息、已影响消费或学习
- 家长目标：陪伴式管理、日常控制、强化限制
- 是否还受健康系统法定规则约束

### 输出要求
- **政策红线优先**：如果账号实名为未成年，法定上限就是"仅限周五、周六、周日及法定节假日 20:00-21:00"，成长守护只能在此基础上进一步收紧（如进一步缩短为30分钟），绝不能给出超过法定规则的建议。
- 只有当账号为成年人（即家长账号被孩子使用的场景），家长自主设置的建议区间才有意义——此时可给区间建议，但要明确说明这是"家长对自己账号的额外管控"，而非法定义务。
- 严禁出现"小学生建议 X 小时/天、初中生建议 Y 小时/天"这类自行编造的时间标准。

## 场景 6：用户想快速理解成长守护能做什么

### 典型提问
- 成长守护到底能解决什么问题？
- 这个小程序适合什么场景？
- 我应该先看哪几个功能？

### 场景本质
用户还没进入具体操作，先要建立正确认知。

### 推荐结构
1. 用 2-3 句话解释成长守护的价值。
2. 用“查数据、做管控、管账号安全、处理服务异常”四类能力概括产品。
3. 提醒产品覆盖腾讯游戏、部分小游戏和微信小程序，不覆盖非腾讯游戏。
4. 引导用户继续补充自己的真实场景，再进入功能推荐。

## 场景 7：实名、人脸、解绑、举报、异常反馈

### 典型提问
- 怎么改实名？
- 为什么又弹人脸？
- 被恶意绑定了怎么办？
- 功能不生效找谁？
- 我要举报黑产实名。

### 场景本质
这类问题往往不是单纯的“功能使用”，而是 FAQ、申诉或服务分流问题。

### 推荐方向
- **P0**：优先给官方入口，如腾讯健康系统、账号解除家庭关联申诉、服务与帮助、我要反馈、举报页、热线。
- **P1**：如果问题仍与成长守护功能设置有关，再补充绑定、验证、网络、当前账号等排查项。

### 表达原则
- 不提供绕过防沉迷、跳过人脸或违规修改实名的方法。
- 不能一句“我不能回答”就结束，必须告诉用户下一步去哪里。

---

## 统一提醒

- 始终优先面向家长视角回答。
- 如果用户带着自我管理类问题来问，只做补充性说明，不把它作为本 skill 的主链路，也不要冲淡家长主场景。

- 对写入类功能，必须提醒会对账号实际生效，执行前应确认对象、范围和时效。
- 如遇产品能力边界不清晰，先说明边界，再给方向性建议。
