# 去 AI 味规则

> **应用时机**：Step 2 反思 + Step 3 雅化。

## AI 翻译典型 Tell-Tale Sign（要识别 + 消除）

### 模式 1：模板化开篇

```
❌ "在本文中，我们..."（每段都用）
❌ "本研究旨在..."
❌ "首先...其次...最后..."（列表化）
❌ "需要指出的是..."
❌ "值得注意的是..."（句首）
❌ "众所周知..."

✅ 视段落功能精简或删除
✅ "近年来，X 成为..."（仅 Intro 段首）
```

### 模式 2：排比结构滥用

```
❌ "我们提出了 X。我们设计了 Y。我们实现了 Z。"（机械排比）
❌ "X 提供了... Y 提供了... Z 提供了..."

✅ 合并："我们提出 X、设计 Y、实现 Z"
✅ 重组："X 是核心贡献，配合 Y 和 Z 实现了..."
```

### 模式 3：空洞填充

```
❌ "在很大程度上"
❌ "在某种程度上"
❌ "在一定意义上"
❌ "总体来说" / "总的来说"
❌ "毋庸置疑"
❌ "众所周知"
❌ "不言而喻"

✅ 用具体证据替换或删除
```

### 模式 4：被动滥用（中文）

中文不像英文那样大量使用被动语态。AI 翻译常把英文被动直译为中文被动，读着别扭：

```
❌ "该方法被提出于 2020 年"
❌ "实验被进行了三次"
❌ "该模型被证明是有效的"

✅ "该方法于 2020 年提出"
✅ "实验进行了三次" / "我们进行了三次实验"
✅ "实验证明该模型有效"
```

### 模式 5：句子结构镜像（直译腔）

```
❌ "该工作的贡献被总结如下" → AI 直译 "The contributions of this work are summarized as follows"
✅ "本文贡献如下" / "本文的主要贡献包括"

❌ "众多研究已经被进行" → "Many studies have been conducted"
✅ "已有诸多研究" / "前人开展了大量研究"
```

### 模式 6：段落开头都是"我们"

```
❌ "我们提出..." / "我们设计..." / "我们实现..." / "我们测试..."（每段开头）

✅ 偶尔换主语：
   - "本文方法..."
   - "实验结果..."
   - "如图 1 所示，..."
   - "为了 X，我们..."
```

### 模式 7：列表化叙述

```
❌ "首先，X 是 A。其次，Y 是 B。再次，Z 是 C。最后..."
（4-5 个"首先/其次/再次/最后" 在一段中）

✅ 用自然语言连接：
   "X 是 A；与之类似，Y 是 B。Z 则不同，它..."
```

### 模式 8：过度修饰（形容词堆砌）

```
❌ "我们提出了一种新颖的、有效的、高效的、强大的方法"

✅ 选最关键的 1-2 个：
   "我们提出一种高效方法，在 X 上取得 Y 的提升"
```

## 句法层面的 AI 味

### 1. 句长方差小

AI 倾向输出"中规中矩"的句子，长度都在 15-25 字之间，**人类写作有 5-50 字的自然方差**。

修正：刻意混合长短句，短句强调结论，长句承载论证。

### 2. 标点单调

AI 几乎只用句号和逗号。**人类学术写作适度用 ; — 等**。

修正：适度引入分号（连接相关从句）、破折号（插入说明）、冒号（引出列表或定义）。

### 3. 段落首句模板

```
❌ 每段第一句都是"在 X 章节中，我们将讨论..."

✅ 用主题句直接进入主题
```

## 词汇层面的 AI 味

| AI 偏好 | 人类替代 |
|---|---|
| "深入" / "全面" / "完整" | （删除或具体化） |
| "强大的" / "卓越的" | （改为具体证据） |
| "本质上" / "从根本上" | 视语境删除 |
| "至关重要" / "极其重要" | （改为具体说明 X 重要在何处） |
| "如下所述" / "如上所述" | （直接说所述内容，不用元指） |
| "鉴于" / "鉴于此" | "因此" / "为此" |

## 中→英时的 AI 味识别

英文 AI 翻译的特征词（要避免过度使用）：

- "delve into" — AI GPT 标志词
- "navigate" / "navigating"（非地理意义）
- "embark on a journey" / "journey"（非旅行意义）
- "in the realm of" / "in the world of"
- "tapestry" / "vibrant"（非视觉描述）
- "groundbreaking" / "revolutionary"（非具体证据）
- "leverage"（动词时滥用）
- "showcase"（动词时滥用）
- "highlight"（动词时滥用，"指出"即可）

## 校验启发式

Step 2 反思时，用以下信号检测段落是否有 AI 味：

```python
def detect_ai_flavor(paragraph_zh):
    flags = []
    if paragraph_zh.startswith(("在本文中", "在本研究中", "本研究旨在")):
        flags.append("opening_template")
    if paragraph_zh.count("我们") > 5 and len(paragraph_zh) < 200:
        flags.append("we_overuse")
    if paragraph_zh.count("被") > 3 and len(paragraph_zh) < 200:
        flags.append("passive_overuse")
    if "首先" in paragraph_zh and "其次" in paragraph_zh and "最后" in paragraph_zh:
        flags.append("listicle")
    fluff_words = ["在很大程度上", "在某种程度上", "众所周知", "毋庸置疑", "需要指出的是", "值得注意的是"]
    for fluff in fluff_words:
        if fluff in paragraph_zh:
            flags.append(f"fluff:{fluff}")
    return flags
```

任一 flag → 进入"去 AI 味重写"子流程：
1. 列出 flag
2. 引导 Step 3 雅化时优先消除这些 flag
3. 重写后回扫，确保 flag 数量下降

## 不要矫枉过正

下列**不是** AI 味，是**正确的学术写作**：

- ✅ "我们提出..." — 学术允许第一人称复数
- ✅ "实验表明..." — 客观陈述
- ✅ "本文贡献如下" — 适度元指
- ✅ "由表 1 可见..." — 数据引用
- ✅ "如算法 1 所示..." — 流程引用

去 AI 味的目的是**消除模板化**，而不是排斥所有学术套话。
