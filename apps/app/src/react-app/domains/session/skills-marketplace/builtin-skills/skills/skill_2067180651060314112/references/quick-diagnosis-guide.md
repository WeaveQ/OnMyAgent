# Quick Diagnosis Decision Guide

快速诊断不是简单评价好坏，而是先判断：

> **这个文本现在处于哪个开发阶段，最适合先做什么。**

它是所有正式模块的入口。

---

## 快速诊断必须回答的核心问题

1. **故事核心是否清楚？**
   - 能否用一句话说清“谁想要什么，但因为什么得不到 / 不敢要 / 不能要”。
2. **人物是否成立？**
   - 角色的欲望、恐惧、防御、行动是否存在。
3. **关系或冲突是否有推进？**
   - 不是事件堆叠，而是人物状态是否变化。
4. **是否有可影视化场景？**
   - 是否有具体空间、动作、物件、声音、光线，而不只是概念说明。
5. **是否有潜台词与母题？**
   - 人物是否只在说信息；是否已经出现可承载情绪的物件 / 动作 / 空间。
6. **终局或阶段目标是否自然？**
   - 结尾是否从人物和结构里长出，而不是作者硬拧。
7. **文本是否收束？**
   - 故事的核心矛盾或人物关系是否走到了一个阶段性终点，还是停在半路。
8. **是否存在"没写完"或"该停没停"的信号？**
   - 文本结尾是自然的收束点，还是中断、弃坑、或者情绪张力已过后的无意义延展。

---

## 诊断等级

### A｜可直接进入完整开发读本

特征：
- 人物动机明确；
- 关系/冲突推进清晰；
- 有多个关键场景；
- 有可回收的物件或动作；
- 终局或阶段终点自然；
- 细节足够支撑演员 / 导演 / 编剧分析。

推荐：
- 先做 **完整开发读本**；
- 再按目标拆演员读本 / 导演视听读本 / 制片摘要 / AI 提示包。

---

### B｜有开发价值，但需补局部信息或先补一层

特征：
- 核心成立；
- 主要关系或冲突成立；
- 但缺少部分关键场景、物件回收、人物厚度、身体逻辑、场景承载或终局清晰度；
- **或文本存在"该停没停"的过度延展**：核心命题已收束，但后续有若干无推进轮次，用户角色出现重复性日常描写，情绪张力明显下降。

推荐：
- 先做 **编剧改编读本** 或 **补强型演员读本**；
- 不建议一上来全套输出。

如果诊断发现文本有明确的"有效终点"但被延展覆盖：
- 在诊断中指出该有效终点的位置；
- 建议以该节点作为阶段终点进行评估；
- 自然衔接到编剧改编读本的开发路径判断。

输出时要指出：
- 当前最该补哪一项：人物厚度 / 身体逻辑 / 场景承载 / 物件线 / 有效终点截断。

---

### C｜开发入口未定，先做路径判断

特征：
- 文本本身可能很有价值；
- 但不是一个能直接切进去分析的清晰单元；
- 常见于群像长篇、多主角并行、章节式成长小说、强气质文本；
- **或文本明显未完成**：故事核心矛盾尚未收束，人物关系停在半路，没有阶段性终点。

推荐：
- 先做 **编剧改编读本（开发路径判断版）**；
- 或先做 **制片摘要** 判断形态；
- 暂不建议直接整本做完整开发读本。

如果诊断发现文本未完成：
- 明确告知用户"当前文本看起来还没有写完"；
- 建议先补写一个阶段性结局或悬念节点，然后重新诊断；
- 不强行分析一个没有收束的段落。

一句话：
> 不是不能开发，而是开发入口不能选错。

---

### D｜当前叙事尚未成立，需重构

特征：
- 只有概念，没有人物；
- 只有情绪，没有行动；
- 只有事件梗概，没有人物选择；
- 角色前后不一致；
- 缺少可拍场景；
- 关键情绪全靠旁白解释；
- 结局与前文无因果关系。

推荐：
- 只做 **开发诊断 / 最小重构方案**；
- 不建议进入演员读本、导演读本或 AI 提示包。

一句话：
> 现在不是拆读本的时候，而是先让它长成故事的时候。

---

### D0｜不可开发，终止

特征：
- 未成年人性侵犯或性剥削；
- 药物 / 酒精胁迫下的非自愿性行为；
- 奴役 / 人口贩卖的正面描写或美化；
- 角色仅作为欲望发泄对象，无人物动机、无叙事结构；
- 其他不具开发正当性的红线内容。

处理：
- 立即终止；
- 不生成任何读本；
- 只输出终止说明。

---

## 文本收束状态检测

快速诊断不只判断故事好不好，还判断**故事停得对不对**。

很多普通用户没有影视化训练，不知道故事应该在哪里停住。长会话尤其容易产生两类问题：

- **没写完**：矛盾还在、关系还在走、没有阶段性终点；
- **该停没停**：核心命题已收束，但用户舍不得停，又推了几个无推进的轮次。

诊断时必须检查文本收束情况，给出以下判断之一：

| 收束状态 | 说明 |
|:---|:---|
| **已收束** | 有明确的阶段性终点或终局，可以正常进入后续模块 |
| **阶段收束** | 在某个节点形成了有效的闭环节点，但后续仍有延展；建议以该节点作为评估起点 |
| **未收束** | 核心矛盾或人物关系尚未走到任何终点；不建议直接进入完整开发读本 |

### 检测"未完成"的信号（倾向 C 级）

检查以下信号，命中 2 条以上则倾向判为未完成：

1. **最后一条消息是用户的提问、指令或行动推进**，而不是状态描述或结语；
2. **对话中出现"下次再写""先停一下""今天先到这里"等中断性用语**；
3. **故事中明确埋下了冲突或矛盾，但在文本结束处没有被任何形式回应**；
4. **没有任何收束迹象**——不一定是大团圆或悲剧，但没有任何阶段性的情绪或命题闭环。

### 检测"该停没停"的信号（倾向 B 级 + 引导）

检查以下信号，命中 2 条以上则倾向判为过度延展：

1. **在某个关键事件或情绪高潮之后，又延续了若干无推进的轮次**；
2. **用户角色开始出现重复性动作或日常性描写，明显在读条**；
3. **核心命题在情绪或思想上已经完成了闭环**；
4. **最后若干轮次的对话强度、信息密度、情绪张力明显下降**。

如果检测到过度延展，诊断中应：
- 标注"有效终点"的位置（在原文中的大致位置或事件描述）；
- 建议以该终点作为阶段评估节点；
- 自然衔接到编剧改编读本，而不是把多余的尾巴也塞进分析。

## 不足时，必须给明确建议

不要只说“不够好”。要指出最小可执行补法。

### 如果缺人物动机

建议补：
- 角色最想要什么；
- 最怕什么；
- 为什么怕；
- 他/她为了避免恐惧，会做出什么防御动作。

### 如果缺场景

建议至少补 3 个场景：
1. 第一次暴露人物性格的场景；
2. 关系/冲突第一次变化的场景；
3. 人物必须做选择的场景。

### 如果缺人物厚度

建议补：
- 角色反复做的选择；
- 角色稳定的说话方式；
- 面对压力时的第一反应；
- 别人如何看这个角色；
- 一个能暴露他/她生存方式的动作或场景。

### 如果缺身体逻辑

建议补：
- 站、坐、走、看人的方式；
- 手上会做什么小动作；
- 情绪上来时身体先怎么反应；
- 角色与物件、空间的关系。

### 如果缺物件线

建议补一个能反复出现并回收的物件。要求：
- 不要硬象征；
- 能被角色使用、保存、归还、丢弃或改变；
- 最好能在终局回收。

### 如果缺潜台词

建议重写一场对话：
- 表层谈安全的事；
- 底层谈真正的关系或冲突；
- 至少有一句没说出口的话，通过动作表达。

### 如果结尾硬

建议回看：
- 角色是否有能力做出这个选择；
- 前文是否埋过这个选择的种子；
- 是否能用物件/动作代替突然台词；
- 是否需要改成开放 / 悬置 / 阶段性终点。

## 快速诊断额外要补的四个判断

### 1. 当前重心是什么

尽量判断这篇文本更偏：
- 人物
- 关系
- 群像
- 行业
- 设定
- 命题
- 气质

这会直接影响后续推荐模块。

### 2. 当前最危险的误开发方式

指出：
- 最容易把这个文本做坏的方式是什么。

例如：
- 把群像压成单线；
- 把行业剧拍成普通爱情戏；
- 把命题片拍成解释型说教；
- 把强设定文本拍成空心爽文。

### 3. 当前最值得先保住的东西

告诉用户：
- 先别急着全改，最值钱的是什么。

例如：
- 一句台词
- 一个结构
- 一种关系质地
- 一个物件回收
- 一种气味或节奏

### 4. 下一步只推荐 1 个主建议 + 1 个备选

不要给太多模块，让普通用户更容易选。

---

## 快速诊断输出模板

```markdown
# 快速诊断：《标题》

## 1. 诊断等级
A/B/C/D/D0，并用一句话说明。

## 2. 一句话判断
这篇文本现在处于什么开发状态。

## 3. 当前重心
人物 / 关系 / 群像 / 行业 / 设定 / 命题 / 气质（可多选，但要有主次）。

## 4. 故事与人物是否成立
简要判断核心、人物、关系是否成立。

## 5. 当前最强的东西
最多 1-3 条。

## 6. 当前最大的问题
最多 1-3 条。

## 7. 最危险的误开发方式
指出 1 条最容易把它做坏的方式。

## 8. 最值得先保住的东西
指出 1 条当前最值钱的东西。

## 9. 文本收束状态
已收束 / 阶段收束 / 未收束。如果是阶段收束，指出有效终点位置；如果是未收束，说明缺少什么。

## 10. 是否适合生成读本
适合哪种，不适合哪种，为什么。

## 11. 下一步建议
主建议 1 个 + 备选 1 个。
```

---

## 判断语气

要明确，但不要打击用户。

推荐句式：
- “这个故事已经成立，但现在更适合先做编剧改编读本。”
- “现在不建议直接做演员读本，因为角色还没有足够的可表演动作。”
- “这不是不能开发，而是开发入口不能选错。”
- “如果只补一个东西，先补一个能反复出现并在结尾回收的物件。”
