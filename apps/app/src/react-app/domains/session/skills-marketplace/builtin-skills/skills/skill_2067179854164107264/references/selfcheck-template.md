# 剧本自检清单模板

用于 Step 5b：逐稿自检。

## 检查清单

每版自检至少检查以下项目：

### 结构层

- [ ] 是否违反不可改核心（人物关系、结局、关键动机）
- [ ] 是否漏掉必须保留场景
- [ ] 是否有场景功能重复或节奏拖沓
- [ ] 段落比例是否合理

### 内容层

- [ ] 是否漏掉必须保留台词/物件/母题
- [ ] 是否新增未经允许的狗血冲突（第三者、车祸、绝症等）
- [ ] 是否有角色动机跳跃
- [ ] 是否把克制关系写成直白告白

### 格式层

- [ ] 是否台词过白（潜台词被说破）
- [ ] 是否动作说明不可拍（写了不可见心理）
- [ ] 是否格式不统一（场标题、对白格式、微信格式）
- [ ] 是否把剧本写成小说、散文或读后感

### 版本对比（二稿及以后）

- [ ] 本稿相对于上一稿改进了什么
- [ ] 本稿是否引入新的问题

## 修订优先级

自检报告必须给出明确分级：

| 级别 | 含义 | 处理方式 |
|:--|:--|:--|
| **致命问题** | 违反不可改核心、人物关系跑偏、终局被破坏 | 必须修改 |
| **高优先级** | 台词过白、场景遗漏、狗血化、格式严重不统一 | 建议修改 |
| **可优化** | 节奏微调、措辞润色、细节补充 | 可选修改 |

## 进入下一稿的触发条件

- 若自检发现**致命问题**，必须建议用户进入下一稿修订；
- 若自检发现**高优先级问题超过 3 项**，应建议用户进入下一稿修订；
- 若仅有**可优化项**，可以建议用户直接进入分镜，不做三稿。

## 命名规则

- `项目名_标准剧本一稿.md` → `项目名_剧本一稿自检报告.md`
- `项目名_标准剧本二稿.md` → `项目名_剧本二稿自检报告.md`
- 以此类推。

如果用户要求继续下一稿，必须以上一稿自检报告作为修订依据。
