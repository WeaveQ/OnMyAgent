# V1 功能-场景映射表

把"用户怎么描述问题"快速翻译成"该推荐什么功能、先做什么、去哪处理"。

## 使用原则

1. 先判断账号来源：孩子自己的账号 / 家长账号 / 他人账号。
2. 再判断用户意图：理解、查询、限制、校准、异常处理。
3. 先给 P0 主功能，再给 P1 补充功能。
4. 写入类动作必须提醒二次确认；高风险问题必须分流官方入口。

## 一、总表：场景 → 功能

| 场景 | 典型提问 | P0 主功能 | P1 补充功能 | 前置条件 | 需要提醒 |
|---|---|---|---|---|---|
| 不知道孩子为什么还能玩 | 为什么还能玩、国家不是有限制吗 | 管控状态解释 + 账号来源判断 + 游戏时间限制/充值限制 | 身份证名下账号查询、绿色手机上报、人脸识别管控、家长服务助手 | 先判断孩子用谁的账号 | 不要直接假设是功能失效，先排查账号和实名 |
| 孩子用自己账号，家长想开始管 | 我想管孩子、怎么绑定孩子 | 家庭管理绑定 + 孩子身份验证 + 时间限制/充值限制 | 游戏禁玩、小程序管理、周报 | 需要绑定成功；如要管控还需孩子身份验证 | 家长绑定后仅查询与查询+管控的门槛不同 |
| 孩子用家长账号玩 | 孩子拿我手机玩、怎么防偷玩 | 人脸识别管控 / 自助下发人脸识别 | 临时锁/游戏临时开关、绿色手机上报、守护锁 | 需家长身份核验 | 这是"防冒用"场景，不一定先推家庭绑定 |
| 孩子用他人账号玩 | 用爷爷奶奶手机、我不知道什么号 | 绿色手机上报 | 身份证名下账号查询、疑似小号排查、家长服务助手 | 知道对方账号/微信更容易推进 | 不知道账号时要先说明边界，不能承诺直接管住 |
| 想临时让孩子玩一阵，但要可控 | 先玩半小时、今晚让他玩一会儿 | 临时锁 / 游戏临时开关 / 临时放行 | 消费限制、守护锁、人脸识别管控 | 需确认对象、时长、作用范围 | 临时放行可能覆盖原有限制，到期规则要说明 |
| 想做日常稳定管理 | 平时怎么设置比较合适 | 游戏时间限制 + 游戏充值限制 | 游戏禁玩、小程序管理、游戏周报、提醒设置 | 最好已绑定并完成身份验证 | 建议给区间，不给唯一数字 |
| 已经发生偷玩/乱充/夜间玩 | 孩子半夜玩、偷偷充钱 | 游戏禁玩 / 一键强限制 + 充值限制 | 守护锁、人脸识别管控、周报 | 需明确是哪个账号在玩 | 先止损，再谈长期方案 |
| 想看最近有没有玩、玩了多久 | 今天玩了多久、近7天玩了哪些游戏 | 游戏时长查询 | 游戏周报、登录明细 | 家长查孩子需已绑定 | 数据通常以近30天为主 |
| 想看充值情况 | 本月充了多少、昨天有没有充值 | 充值记录查询 | 游戏周报、消费限制 | 家长查孩子需已绑定 | 如需设置限额要单独二次确认 |
| 怀疑孩子冒用了家长身份/账号 | 我的身份证下有哪些号 | 查询身份证名下游戏账号 | 人脸识别管控、绿色手机上报 | 强身份核验 | 涉及敏感信息，不能跳过验证 |
| 想限制某一款游戏 | 不让玩王者、和平精英禁掉 | 游戏禁玩 | 时间限制、周报 | 需确认具体游戏和时效 | 单游戏禁玩和全局限时要区分 |
| 想管理微信小程序 | 怎么管小程序、限制某个小程序 | 微信小程序管理 | 总时间管理、单款管理、禁用 | 需家长绑定并验证 | 小程序管理和游戏管理要分开说 |
| 功能不生效 / 找不到入口 | 为什么没生效、按钮在哪 | 先按账号/绑定/验证/网络排查 | 家长服务助手、热线、服务与帮助 | 需先判断是设置问题还是服务问题 | 不要一开始就推热线 |
| 实名 / 人脸 / 举报 / 解绑 / 黑产 | 改实名、被误绑、我要举报 | FAQ/官方入口分流 | 服务与帮助、账号解除家庭关联申诉、腾讯健康系统、热线 | 视问题类型而定 | 不提供绕过或规避方法 |

## 二、按账号场景细化推荐

### 1. 孩子用自己的账号
- **链路**：家庭管理绑定 → 查询绑定可查数据 → 补孩子身份验证后可设管控
- **P0**：家庭管理、时间限制、充值限制
- **P1**：游戏禁玩、小程序管理、游戏周报、守护锁

### 2. 孩子用家长账号
- **链路**：确认是家长登录账号 → 防冒用优先 → 临时控制 → 防篡改
- **P0**：人脸识别管控 / 自助下发人脸识别
- **P1**：临时锁/临时开关、守护锁、绿色手机上报

### 3. 孩子用他人账号
- **链路**：判断是否知道账号归属 → 知道则绿色手机上报 → 不知道先说明边界再排查
- **P0**：绿色手机上报
- **P1**：身份证名下账号查询、家长服务助手、疑似小号排查

## 三、按问题意图细化推荐

### 1. 查询类
- **时长**：游戏时长查询
- **充值**：充值记录查询
- **总览趋势**：游戏周报
- **名下账号**：身份证名下账号查询
- **小程序记录**：微信小程序记录查询

### 2. 管控类
- **长期限时**：游戏时间限制
- **长期限充**：游戏充值限制
- **指定游戏不让玩**：游戏禁玩
- **临时放行**：临时锁 / 临时开关
- **防冒用**：人脸识别管控 / 自助下发人脸识别
- **防篡改**：守护锁
- **把他人微信纳入规则**：绿色手机上报
- **管微信小程序**：微信小程序管理

### 3. 服务类
- **绑定/解绑/误绑**：家庭管理、账号解除家庭关联申诉
- **复杂问题升级**：家长服务助手、家长柜台
- **反馈/举报**：服务与帮助、我要反馈、防沉迷举报
- **健康系统/实名问题**：腾讯健康系统

## 四、优先级与冲突口径

### 1. 临时放行相关
- **游戏临时开关 / 临时放行**的优先级可以高于既有禁玩或限时。
- 回答时要明确：临时规则到期后是否恢复原方案，需要向家长说明。

### 2. 禁玩 vs 限时
- **限时**：更适合控制每天时长或时段。
- **禁玩**：更适合禁止某个游戏，或高风险止损场景。
- 用户说"别让他今晚再玩了"，要先判断是"剩余时长设为0"还是"禁掉某一款游戏"。

### 3. 家庭绑定 vs 家长账号防冒用
- 孩子用自己账号：优先家庭绑定。
- 孩子用家长账号：优先防冒用工具，不一定先推绑定。
